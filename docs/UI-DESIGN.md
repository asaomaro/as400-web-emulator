# UI デザインガイド（web-ui）

このセッションの UI 調整で確立した、**再利用すべき見た目・振る舞いの規約**をまとめる。
新規の画面・コンポーネントを足すときはこの規約に合わせる（`packages/web-ui`）。

---

## テーマ / 配色

- **配色は CSS 変数で一元化**し、コンポーネントは変数を使う（生の色コードを避ける）。定義は
  `packages/web-ui/src/styles.css`（light / dark を両方定義）。
- 主要変数:
  - 面・枠: `--crt`（CRT 面）/ `--crt-bezel` / `--crt-line`、`--card`、`--line`、`--bg`
  - 文字: `--ink`（本文）/ `--muted`
  - アクション・状態: `--accent`（**緑・主アクション/プリンター**）/ `--accent-soft`、
    `--t-green` / `--t-blue`（**5250端末**）/ `--t-yellow` / `--t-red`
- **テーマは `useTheme`**（`composables/useTheme.ts`）: ライト → ダーク → **システム（OS 追従）** の 3 モード。
  `data-theme` 属性で適用し、system は `prefers-color-scheme` 変更に追従する。
- **新規スタイルは light / dark 両テーマで確認**する（変数経由なら基本追従する）。

## ボタン意匠（面の系統に合わせる）

用途・設置面で意匠を統一する。混在させない。

- **トップバーのトグル** `.theme-btn`: `--card` 背景・`--line` 枠・角丸。**固定高さ 28px＋`inline-flex` 中央寄せ**
  （絵文字の有無で高さがブレないように）。
- **CRT ペイン内ツールバー**（プリンターペイン等の暗色面）: `StatusBar` の `.fk` に合わせる
  = `--crt` 背景・`--crt-line` 枠・`--muted` 文字・hover で `--t-green`。
- **テキストリンク** `.link`: 枠/背景なし・`--accent` 色。

## トグルボタンの安定化（レイアウトシフト防止）

> **鉄則: トグルボタンは ON/OFF（状態切替）でボタンの幅も高さも変わらないこと。**
> 自分が動くだけでなく、**隣接するボタンの位置がずれる**のが実害。新しいトグルを足すときは必ず満たす。

- **幅**: 切替でラベル幅が変わる部分（`ON`/`OFF`、`カナ`/`英`、`{ }`/`␣`、`☀ ライト`/`🖥 システム` 等）は、
  **固定幅の `inline-block` span で幅を確保**して、ボタン幅を一定に保つ。幅は**長い方の状態に合わせる**。
  ```html
  <!-- 良い例: 変化する部分だけを固定幅 span で包む -->
  自動出力: <span class="onoff">{{ enabled ? "ON" : "OFF" }}</span>
  <!-- .onoff { display: inline-block; width: 2.2em; text-align: left; } -->
  ```
  ラベル全体が入れ替わる場合（`☀ ライト` ⇄ `🖥 システム`）は**ラベル全体を固定幅 span で包む**。
- **高さ**: 絵文字の有無で行高が変わるため、`height` を固定し `inline-flex` + `align-items:center` で中央寄せする
  （トップバーは 28px）。`line-height: 1` を併用する。
- 複数トグルは**1 グループ（`.toggles`）にまとめ、`margin-left:auto` はグループに 1 つだけ**付ける。
  個々のボタンに `margin-left:auto` を付けると flex で均等分散して**バラつく**（アンチパターン）。
- アイコングリフは**同サイズの対**を使う（例 `«`/`»`、`◧`/`◨`）。`◧`/`▤` のように別ブロックの
  異サイズ文字を対にしない（開/閉でサイズが変わる）。

## 情報ポップオーバー

- 構造: **バックドロップ**（`position: fixed; inset: 0;` 透明）＋**本体**（`position: absolute; top: 100%;`。
  アンカーとなる親に `position: relative`）。共通実装は `InfoPopover.vue`（`rows` を渡す）。
- **閉じ方**: バックドロップ（外側）クリック＋トリガ再クリック。本体は `@click.stop` / `@mousedown.stop` で
  伝播を止める。
- **行のラベル/値**: ラベル列は全角ラベルが折り返さない幅＋`white-space: nowrap`。長い値（FQDN 等）は
  `word-break` 許可、ジョブ番号のような一体データは `nowrap`（`SessionInfo.vue` 参照）。

## セッション種別の表記・チップ

- 種別ラベルは **「5250端末」（表示セッション）/「プリンター」** を使う。**「表示」「エミュレーター」は使わない**
  （アプリ全体がエミュレーターのため紛らわしい。「5250端末」がプリンターと対になり明確）。
- チップ（`.kind`）配色で種別を一目化: **5250端末＝青（`--t-blue`）/ プリンター＝緑（`--accent`）/ サーバー設定＝破線**。

## リスト / カード表示

- 表示モードは**専用の修飾クラス**（`view-card` / `view-list`）で分岐する。
  **`.list.list` のような同一クラス重複セレクタは使わない**——CSS では「クラスを 1 つ持つ要素」と同義になり、
  意図せず両モードに当たる（実際にカード表示が一覧レイアウトで崩れた原因）。
- 一覧モードは 1 行の高さを抑える（アクションアイコンは横並び＝`flex-direction: row`、`flex: none`）。

## ローディング表示

- 接続待ち等の待機は**全面オーバーレイ＋スピナー**を出す（`.spinner` + `@keyframes spin`）。
  `@media (prefers-reduced-motion: reduce)` でアニメーションを緩める。`role="status"` を付ける。

## アクティブ状態に応じた表示切り替え

- ヘッダー等の**機能ボタンは、アクティブ（フォーカス中）ペインの内容に連動**させる。ペイン分割があるため、
  `workspaceStore.focusedGroup().activeTab` のセッション種別で判定する。
  - 例: 5250 専用トグル（SO/SI・カナ・リンク・キー）は**表示セッションがアクティブなときだけ**出す。
    接続画面・`admin:*` タブ・プリンター・空ペインでは隠す。テーマ切替は常時表示。

## ペインの最大化

- **分割しているときだけ**タブ帯の右端に最大化ボタンを出す（`PaneTabs` の `.maximize`）。
  単一ペインは既に全面なので、押しても何も変わらないボタンは置かない。最大化中は同じ位置が「元に戻す」になる。
- 最大化は `workspaceStore.maximizedGroupId` の**表示上の切り替えだけ**で、分割ツリー（`root`）は書き換えない。
  比率も配置もそのまま残るので、元に戻すのは ID を消すだけで済む。描画は `displayRoot()` が担う。
- **最大化中はさらに分割できない**（入れ子になり、戻したときの形が予測できなくなる）。端ドロップは
  `zoneFrom` が無効化し、`split()` は合流（`moveTab`）へフォールバックする。**タブ移動だけ許可**する。
- グループが消える／分割が解けたら `syncMaximized()` が最大化を自動解除する。

## 一覧表の列見出し

- 表を持つペイン（一覧・管理・SQL）は**ペイン自体をスクロールさせない**。ペイン根を `flex-direction: column` にし、
  スクロールは内側の 1 枚（`.scroll` / `.section` / `.rows-scroll`）に閉じ込める。
  こうすると絞り込みの帯や見出しが流れず、`thead th { position: sticky; top: 0 }` が**ペインの上端**に効く。
- sticky な `th` には**背景色（`--card`）が必須**——行に背景が無いので、無いと本文が透けて重なる。
- `border-collapse: collapse` では sticky にした `th` の罫線が一緒にスクロールして消える。
  罫線は `border-bottom` ではなく `box-shadow: inset 0 -1px 0 var(--line)` で描く。

## SQL フッターの接続表示

- SQL ペインのフッターには、**いま繋がっているホスト側のジョブ**（`job=…`）と接続先（`host:port`）を出す。
  5250 の OIA と同じで「どこに繋がっているか」は常に見えるところに置く（従来は実行ログを開かないと分からなかった）。
- 実行ログの `connect` 行は**張り直したときだけ**出すが、フッターは**使い回し（`reused`）でも更新する**——
  ログは「いつ張り直したか」、フッターは「いまどれか」を答えるものなので、条件が違う。

## 信頼設定の UI ゲート

- サーバー上の**パス書き込み・コマンド実行・秘密**につながる設定（PDF 出力先 `autoPdfDir`・自動印刷
  `autoPrint`・パスワード等）は、**認証オフ（ローカル単一利用）または admin のときだけ** UI に露出・編集させる。
  サーバー側でも同じゲート（`canEditProfiles`）で受理し、一般ユーザー・未認証からは受理も露出もしない（信頼境界）。

## アクセシビリティ / 動作

- キーボード focus の可視化、`prefers-reduced-motion` の尊重、`role` / `aria-*`（`role="status"`・
  `aria-pressed`・`aria-label` 等）を適切に付ける。
