# レビュー: 自動印刷の宛先を保存時に確認する

## ラウンド 1

### 設計の要点
- **エラーではなく警告**。プリンターは実際に印刷して確かめられず（紙が出る）、CUPS が無い等
  「確認手段が無い」環境もある。確実に判定できないものを保存の可否に使わない。
- 「宛先が無い」と「確認できない」を**区別して伝える**（lp 無し / lpstat 無し / 照会失敗の 3 分岐）。
- **タイムアウト**（既定 3 秒）。ネットワーク宛先で保存操作がブロックされるのを防ぐ。
- PATH 探索は `which` をサブプロセスで呼ばず `fs.access(X_OK)` で行う（起動コストと移植性）。

### テストの環境非依存化
実環境の lp/lpstat の有無に結果が左右されると、CI と手元で挙動が変わる。PATH を差し替えた
**偽コマンド**（`#!/bin/sh` + exit code）で 5 分岐すべてを固定した。

- 実装中、偽 lpstat の `sleep 5` が PATH 差し替えの影響で見つからず即終了し、タイムアウトのつもりが
  「見つからない」で通っていた。`/bin/sleep` の絶対パスに直して修正済み（テストが嘘をつくところだった）。

### 非対称の妥当性
`autoPdfDir`=エラー / `autoPrint`=警告 という非対称は残るが、これは「確実に判定できるか」の違いを
反映したもので恣意的ではない。README に理由を明記した。

### 指摘
- [nit] プリンターの状態（停止中・ジョブ拒否中）は見ていない。宛先が存在しても印刷が失敗する余地は残る。
  受信時の警告・ステータス表示で拾えるため、保存時にそこまで踏み込まない判断。**未対応（意図的）**。
- [nit] Windows には `lp` が無く、この機能自体が動かない。警告文でそれが分かる（「lp コマンドが
  見つかりません」）ため、OS 判定は入れていない。

### 判定
