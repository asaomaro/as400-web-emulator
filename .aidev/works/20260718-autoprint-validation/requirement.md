# 要件: 自動印刷の宛先を保存時に確認する（警告）

## 背景
`autoPdfDir` は PR #57 で保存時に検証されるようになったが、`autoPrint`（プリンター名）は素通りで、
タイポしても保存でき、**スプール受信時に `lp が異常終了しました` で初めて分かる**。
#57 で解消したはずの「設定した場所・時点から離れて失敗する」問題が印刷側に残っていた。

## 目的
宛先の問題に**保存時に気づける**ようにする。ただし出力先と同じ強さでは判定できないため、
**エラーではなく警告**として扱う。

## スコープ
### 対象
- 保存時に `lp` の有無を確認する（無ければ「この環境では自動印刷が動きません」）。
- `lpstat -p <名前>` で宛先を照会する（引けなければ「見つかりません」）。
- いずれも**警告**として応答に含め、UI に出す。**保存は成功させる**。
- ネットワーク宛先で保存操作がブロックされないようタイムアウトする。

### 対象外
- プリンターの**状態**（disabled / rejecting）の確認。
- 保存を止めること（エラー化）。
- Windows 対応（`lp` 前提の機能）。

## 完了条件
- [ ] `lp` が無ければ警告が出る。
- [ ] `lpstat` が無ければ「確認できない」と分かる警告が出る（「見つからない」と区別）。
- [ ] 宛先が引ければ警告が出ない。
- [ ] 宛先が引けなければ警告が出るが**保存は成功**する。
- [ ] 応答が遅い宛先で保存がブロックされない。
- [ ] 出力先の不備は従来どおりエラー（400）のまま。
- [ ] README に非対称の理由が書かれている。
