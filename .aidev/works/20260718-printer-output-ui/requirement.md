# 要件: PDF 自動蓄積・自動印刷の UI 設定（信頼ユーザー限定）

## 背景 / 課題
プリンター出力設定（PDF 自動蓄積 `autoPdfDir` / 物理自動印刷 `autoPrint` など）は、任意パス書き込み・`lp`
実行につながる「信頼設定」として、現状は運用者管理の `profiles.local.json` の `printer` ブロックでしか設定できず、
**画面からは設定できない**。ローカル単一利用者や管理者にとっては、ファイルを直接編集する必要があり不便。

## 目的 / ゴール
**認証オフ（マルチユーザーでない）、または admin ユーザー**のときに限り、共有プロファイルの printer 出力設定を
**接続画面から編集**できるようにする（＝プロファイル編集と同じゲート）。信頼境界は「書き込みを許可されるのは
その信頼ユーザーのみ」という形で維持する。

## スコープ
### 対象
- プロファイル編集フォーム（既に auth-off/admin で編集可）に **printer 出力設定**の入力欄を追加:
  - `autoPdfDir`（PDF 自動蓄積フォルダ）
  - `autoPrint`（自動印刷プリンター名）
  - `pdfFontPath` / `pdfFontName` / `pageSize` / `fontSize`（PDF 体裁・任意）
- サーバー: これらを**編集可能ユーザー（canEditProfiles = 認証オフ or admin ＆ ファイル由来）からのみ**受理し、
  `profiles.json` に保存。編集不可ユーザー・未認証は従来どおり不可（信頼境界維持）。
- 編集フォームのプレフィルのため、printer 設定を **編集者にのみ**返す（一般公開の一覧には出さない）。

### 対象外
- printer 設定を「ユーザー接続（connections）」側に持たせること（信頼設定はプロファイル限定を維持）。
- 認証オンの一般ユーザーが printer 出力を設定できるようにすること（不可のまま）。
- 出力先パスの実在検証・書き込み権限の事前チェック（受信時に既存の挙動＝失敗は warn で degrade）。

## 機能要件
- auth-off または admin のとき、プロファイル編集フォームで printer 出力設定を追加・変更・クリアできる。
- 保存すると `profiles.json` の該当プロファイル `printer` に反映され、以後のプリンターセッションで有効になる。
- printer 設定が無い/空ならブロックごと保存しない（従来の「未設定＝無効」と同じ）。

## 非機能要件 / 制約
- **信頼境界の維持**: printer 出力設定を受理するのは canEditProfiles を満たすリクエストのみ。認証オンの一般ユーザー・
  未認証からは受理しない（型・ルート双方で担保）。
- `autoPrint` は `spawn("lp", ["-d", <name>, file])`（シェル非経由）で、プリンター名からのコマンド注入は不可
  （現行実装のまま）。`autoPdfDir` はサーバーが PDF を書き込むフォルダ（生成ファイル名は固定パターン）。
- 既存の web-ui テスト・サーバーテスト・ビルド（vue-tsc/Vite）を壊さない。

## 完了条件 (受け入れ基準)
- [ ] auth-off または admin で、プロファイル編集画面から autoPdfDir/autoPrint 等を設定・保存でき、
      `profiles.json` に反映される。
- [ ] 認証オンの一般ユーザー・未認証は printer 設定を編集・注入できない（403/400 または非表示）。
- [ ] printer 設定は編集者にのみ API 露出され、一般公開の一覧には出ない。
- [ ] 空にするとブロックが消え、自動蓄積/印刷が無効化される。
- [ ] 受信スプールが指定フォルダへ PDF 保存され、指定プリンターへ自動印刷される（従来のサーバー機能が UI 設定でも動く）。

## 未確定事項 / 確認したいこと
- **Q1（表示条件）**: printer 出力欄はプロファイルの種別が printer のときだけ出すか、常に出すか。
  （プロファイルの sessionType は printer 設定の有無で決まるため、鶏卵。→ spec で整理）
- **Q2（パスの扱いの注意喚起）**: UI に「サーバーのローカルパス・サーバー上のプリンター名を指定する」旨の注意書きを
  出すか（誤設定・権限で受信時に失敗し得る）。
