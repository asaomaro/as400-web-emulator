# 要件: core の未適用の負債を返す

## 背景 / 課題

backlog `library-extraction.md` の「先に返すべき負債」。**切り出しの実作業より、こちらが本体**と
backlog 自身が書いている。

とくに重いのは、**同じ提案が 2 回の retro で出て 2 回とも適用されていない**ことである:

- `20260718-acs-data-transfer` の retro:
  「`eslint` に `no-restricted-globals` を追加し、core のピュア層で `Buffer` / `process` /
  `setTimeout` 等の Node グローバルを禁止する。現状は `no-restricted-imports` のみで、
  **グローバル参照という抜け道**が空いている」
- `20260718-hostserver-sql` の retro:
  「`eslint` に `no-restricted-globals` を追加する（**前回の retro でも提案し、未適用**）。
  今回は手作業と review で `Buffer` の混入を防げたが、**仕組みで防げていない**」

2 回とも「人間の注意深さ」で事故を防いだだけで、3 回目に守られる保証がない。
**提案が繰り返されること自体が、返済されていない負債の症状**である。

## 目的 / ゴール

core を「他プロジェクトから使えるライブラリ」に近づける前提を整える。
ライブラリ切り出し（別作業）を**やりやすくする**のが目的であって、切り出し自体はしない。

## スコープ候補（spec で取捨選択する）

backlog に挙がっている 4 件＋ retro の 1 件。**全部やるとは限らない**——
実測してから、価値とリスクで取捨する。

1. **`no-restricted-globals` の追加**（retro 2 回・未適用）
   - core のピュア層で `Buffer` / `process` / `setTimeout` 等のグローバル参照を禁止する
   - 現状は `no-restricted-imports` のみで `node:*` の import は塞げているが、
     **グローバル参照は素通し**
2. **ロガーを注入可能にする**
   - 現在 `pino` を直接依存。**ライブラリが利用側にロガーを強制している**
   - backlog に「`hostserver/` は `log.ts` を 1 箇所で使うだけなので、今なら数行で済む」とあるが、
     **これは未検証の見積もり**。実測してから判断する
3. **`Tn5250Error` → `As400Error` 改名**（旧名は別名で維持し既存コードを壊さない）
   - ホストサーバーは TN5250 ではないのに `Tn5250Error` を投げていて、名が体を表していない
4. **`ErrorCode` の整理**
   - 19 種に `FIELD_PROTECTED` / `KEYBOARD_LOCKED` / `NEGOTIATION_TIMEOUT` など
     ホストサーバーに無縁のものが混在
5. **CCSID テーブルの同梱単位の見直し**
   - CCSID 37 の 174 行のために DBCS 込み 18,900 行が付いてくる

## 要検討（spec で決める）

### スコープの取捨 ★最重要

5 件すべてを 1 PR に詰めると、**性質の違うものが混ざる**:

- 1・3 は機械的で低リスク（lint 追加／別名つき改名）
- 2 は API 変更を伴う（利用側の呼び出しが変わりうる）
- 4 は**破壊的変更になりうる**（`ErrorCode` は公開型で、web-ui / server が値で分岐している）
- 5 は**パッケージング/ビルドの変更**で、ブラウザ側の読み込み方に影響する

**リスクの違うものを 1 つの PR に混ぜない**。実測してから、この作業で返すものを決め、
残りは backlog に理由つきで戻す。「全部やる」を目的にしない。

### 後方互換

`Tn5250Error` と `ErrorCode` は **`packages/core/src/index.ts` から export された公開 API**で、
server と web-ui が使っている。**既存コードを壊さない**こと（backlog も「旧名は別名で維持」と書いている）。
