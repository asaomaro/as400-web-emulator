# 要件: 接続・セッション情報表示とプリンターセッションの UX 改善

## 背景 / 課題
接続設定・セッション情報の見え方と、プリンターセッションの運用性に細かな不足がある。
- 接続カードにデバイス名が出ず、設定の全体を一覧で確認できない。
- セッション情報（タブ）の閉じ方が限定的で、接続設定の情報が載っていない。
- プリンターセッションが用紙タイプ問い合わせ（CPA3394）で必ず MSGW 停止し、毎回手動で「I」応答が要る。
- スプール一覧の視認性（サイドバー・フィルタ）と、新着スプールの気づきやすさが不足。

## 目的 / ゴール
接続設定とセッション情報を一貫した情報パネルで確認でき、プリンターセッションが手間なくスプールを
受信・閲覧できる状態にする。

## スコープ
### 対象
1. **接続カード/一覧にデバイス名を表示**する。
2. **接続カード/一覧に情報アイコン（ⓘ）を追加**し、クリックで**その設定の全情報**をポップオーバー等で表示する。
   情報アイコンの再クリック、または**バックドロップ（外側）クリックで閉じる**。
3. **タブ側のセッション情報**（`SessionInfo`）も**バックドロップクリックで閉じる**ようにする。
4. **タブ側セッション情報に接続設定の情報**（ホスト/ポート/CCSID/画面サイズ/デバイス名/TLS/種別/サインオン等）を
   載せる。既存表示（ラベル・状態・画面サイズ・ジョブ等）と**重複する項目は統合**する。
5. **プリンターセッションの CPA3394（MSGW）自動化/改善**：毎回の手動「I」応答を不要にする（実現方法は research）。
6. **プリンターペインの一覧サイドバー開閉**と、**上部のフィルタ入力欄**（スプール名/内容で絞り込み）を追加する。
7. **新着スプール受信をタブで表示**（未読バッジ等）し、該当プリンタータブを見れば気づけるようにする。

### 対象外
- 認証・per-user 分離まわりの仕様変更（既存のまま）。
- MCP ツールのインターフェース変更（表示側の改善が主。必要が生じたら別途）。

## 機能要件
- 接続カード/一覧: デバイス名表示、情報アイコン＋全情報ポップオーバー（アイコン/バックドロップで閉じる）。
- タブ情報: バックドロップで閉じる、接続設定情報を統合表示。
- プリンター: CPA3394 の手動応答を不要化、サイドバー開閉、フィルタ、新着バッジ。

## 非機能要件 / 制約
- 既存の web-ui テスト・ビルド（vue-tsc/Vite）を壊さない。
- プリンターの MSGW 改善はホスト挙動に依存するため、**推測で実装せず research で事実確認**してから spec 化する
  （メモリ方針: ACS/ホスト挙動は聞く・推測しない）。
- 接続設定情報を表示するために、開くセッションへ接続メタ情報を持たせる場合も、資格情報の平文は保持しない。

## 完了条件 (受け入れ基準)
- [ ] 接続カード/一覧にデバイス名が表示される（設定がある場合）。
- [ ] 接続カード/一覧の ⓘ からその設定の全情報が見え、ⓘ 再クリックまたはバックドロップで閉じる。
- [ ] タブのセッション情報がバックドロップで閉じる。
- [ ] タブのセッション情報に接続設定情報が載り、重複項目が統合されている。
- [ ] プリンターセッションで、手動「I」応答なしにスプールを受信・表示できる（または明確に自動化された手順で）。
- [ ] プリンターペインでサイドバーを開閉でき、フィルタ入力でスプール一覧を絞り込める。
- [ ] 新着スプール受信時、該当プリンタータブに未読表示が出る（タブを開くと解消）。

## 未確定事項 / 確認したいこと（research/spec で解消）
- **Q1（MSGW/CPA3394）**【要 research】: 用紙タイプ問い合わせで writer が MSGW 停止する原因と、クライアント側
  （TN5250E プリンターセッション）から手動応答を不要にする実現手段。候補: デバイス作成属性の調整／セッションからの
  自動応答／ホスト側 outq・writer 設定の推奨。PUB400 実機の挙動を事実で確認する。
- **Q2（情報の統合方針）**: タブ情報とカード情報で表示する項目セットと並び順（重複統合のルール）。
- **Q3（新着バッジの解消条件）**: タブを「開く（アクティブ化）」で未読解消か、スプールを閲覧したら解消か。
