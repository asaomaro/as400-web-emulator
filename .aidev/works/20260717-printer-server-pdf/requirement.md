# 要件: プリンターセッションのサーバー側対応（PDF 生成・蓄積・印刷・DL）

## 背景 / 課題
プリンターセッション（PR #15 SBCS / #16 DBCS）はブラウザ内の等幅表示とテキスト保存・印刷→PDF まで。
ユーザーは ACS 相当の「受信スプールを自動で PDF 化して指定フォルダに貯める」「物理プリンターへ自動印刷」を
求めており、これは Node サーバー（コアを保持する側）でしか堅牢に実現できない。

## 目的 / ゴール
サーバー側で受信スプール（帳票）を **SCS→PDF に変換**し、次の 3 つを**プロファイル単位のオプトイン**で提供する:
1. 指定フォルダへの **PDF 自動蓄積**（受信ごとに 1 ファイル）。
2. **物理プリンターへの自動印刷**（OS の印刷経路）。
3. **web-ui / MCP からの PDF ダウンロード**（オンデマンド生成）。
SBCS／DBCS（日本語）の双方で正しく PDF 化できることをゴールとする。

## スコープ
### 対象
- 論理ページ（`LogicalPage[]`）→ PDF 生成（等幅・改ページ保持）。DBCS は CJK フォント埋め込み。
- プロファイル設定に基づく PDF 自動蓄積（出力先ディレクトリ）と自動印刷（プリンター名）。
- PDF ダウンロード: web-ui 向け HTTP エンドポイント＋ MCP ツール。
### 対象外
- SCS の高機能印刷（グラフィックス・バーコード・IPDS）。
- 任意フォントの完全対応（CJK は同梱 or 設定フォント 1 種で足りる範囲）。

## 機能要件
- 受信スプールを PDF に変換できる（SBCS/DBCS、改ページ保持、等幅レイアウト）。
- プロファイルで **PDF 自動蓄積フォルダ**を指定すると、受信ごとに `<dir>` に PDF を保存する。
- プロファイルで **自動印刷プリンター**を指定すると、受信ごとに OS 経由で印刷する。
- web-ui から選択スプールの PDF をダウンロードできる（HTTP）。MCP から PDF（パス or base64）を取得できる。

## 非機能要件 / 制約
- **セキュリティ**: 出力先ディレクトリ・プリンター名は**信頼されたサーバー設定（プロファイル）から**取得し、
  ブラウザの直接指定（WsOpen）では受け付けない（任意パスへの書き込み・任意コマンド実行を防ぐ）。
- 既存の表示・プリンター（受信）経路を壊さない。PDF 機能は追加の任意モジュール。
- CJK フォントは同梱 or 設定で与える（無い場合は SBCS のみ PDF 化し、DBCS は警告して degrade）。
- 実機検証は PUB400（既定）。秘密情報・生成物はコミットしない。

## 完了条件（受け入れ基準）
- [ ] `LogicalPage[]` → PDF（SBCS）が生成でき、改ページ・桁が保たれる。
- [ ] DBCS（日本語）が CJK フォントで PDF 化できる（フォント設定時）。
- [ ] プロファイル指定で受信スプールが指定フォルダに PDF 蓄積される。
- [ ] プロファイル指定で受信スプールが物理プリンターへ自動印刷される（環境が許す場合）。
- [ ] web-ui / MCP から PDF を取得できる。
- [ ] 出力先・プリンターはプロファイル由来のみ（ブラウザ直指定を拒否）。

## 未確定事項（research/spec で解消）
- 採用する PDF 生成ライブラリ（pdfkit 等）と、この環境で追加・利用可能か。
- CJK フォントの入手・同梱可否（サイズ）と、無い場合の degrade 方針。
- OS 印刷経路（`lp`/`lpr`）の有無・呼び出し方（この環境／一般運用）。
- MCP での PDF 返却形式（サーバー保存パス or base64）。
