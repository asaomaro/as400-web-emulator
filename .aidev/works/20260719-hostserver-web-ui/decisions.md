# 決定記録

## D1. ブラウザから任意の SELECT を受け付ける

**前作業（MCP）の D1 を流用していない。** あちらの根拠は「MCP には既に `run_steps` があり
任意 CL を打てる」で、ブラウザ（一般ユーザーを含む）には当てはまらない。
`/api/host/action` が任意 CL を拒んでいるのは、まさにブラウザが対象だからである。

**この作業で新たに取った根拠 — 実機（PUB400）で測った事実:**

`query` が「SELECT 専用」であることは実装の性質としては分かっていたが、
**セキュリティ境界として成立するかは検証していなかった**ので確かめた。

| 投げた SQL | 結果 | 副作用 |
|---|---|---|
| `SELECT 1 FROM SYSIBM.SYSDUMMY1` | 成功 | — |
| `VALUES 1` | 成功 | — |
| `DELETE FROM MARO1.A1 WHERE 1=0` | `PROTOCOL_ERROR` | — |
| `CREATE TABLE MARO1.ZZPROBE (C1 INT)` | `PROTOCOL_ERROR` | **作られていない**（`DLTOBJ` が `CPF2105 not found`） |
| `CALL QSYS2.QCMDEXC('CRTDTAARA …')` | `PROTOCOL_ERROR` | **作られていない**（別経路の一覧で `count=0`） |
| `SELECT …; DROP TABLE …` | `SQL_ERROR` `SQLCODE=-104` | — |

**CALL とデータ域の確認が要点。**「エラーが返った」ことと「実行されなかった」ことは別なので、
**観測可能な副作用を別経路で確かめた**。実装は `prepare + describe` → `open + describe` →
`fetch` の流れで、結果セットを持たない文は describe の段階で落ち、実行には至らない。

**根拠 2**: 読み取り範囲は IBM i の権限が決める（`host-lists.ts:4-6` の既存原則と一貫）。

> ⚠ 断り: 上表は **DDL・CALL・複文について副作用が無いことを確かめた**ものであって、
> 「あらゆる非 SELECT 文が絶対に実行されない」ことの証明ではない。
> **強い経験的根拠として扱い、証明として扱わない。**

### この決定が依存している条件（将来これを壊しうるもの）

**D1 は「`query` が SELECT しか実行できない」という実装の性質に依存している。**
将来 `query` に更新系を通す改造（`executeImmediate` の追加等）を入れると、
ブラウザから任意の更新が通るようになる。依存を見落とさないよう、
この条件は `query.ts` 側ではなく **`host-sql.ts` の冒頭コメントに書いた**
（変更する人が読む場所ではなく、壊れる側に置く）。

## D2. 結果セットの上限はサーバー側で強制する

`maxRows` は既定 200 / 上限 1000 を `z.number().max(1000)` で強制する。
UI の入力欄にも `max` を書いているが、**HTML 属性に強制力は無い**ので境界はサーバーに置く
（AGENTS.md §5「UI の出し分けは補助であって境界ではない」）。

**既知の限界を引き継いでいる**: `query` は結果セットを全件取得してから返すため、
`maxRows` は**応答に載せる行数の上限であって、ホストから取り出す行数の上限ではない**。
UI に常時ヒントを出し、`truncated` のときは警告帯を出して**黙って切らない**。
根本解決は backlog（`stream` の早期打ち切りが未検証のため本作業では触らない）。

## D3. CSV はブラウザ側で生成する

サーバーに CSV エンドポイントを作らない。

**根拠**: サーバーで作ると「表示用に 1 回・CSV 用にもう 1 回」同じ SQL を実行するか、
結果をサーバーに保持するかになる。前者は 1 回 4〜7 秒かかるうえ**2 回目が違う結果を
返しうる**。後者はステートレスを崩す。**画面に出ている表をそのまま落とす**のが
利用者の期待とも一致する。

- **UTF-8 ＋ BOM**: Excel は BOM が無いと UTF-8 CSV をローカルコードページとして開き、
  DBCS（このプロジェクトの中心的関心事）が化ける。実ブラウザで先頭 3 バイトが
  `ef bb bf` であることを確認済み
- **CRLF**（RFC 4180）、エスケープも RFC 4180（`"` → `""`、`,` `"` 改行を含めばクォート）
- 純関数（`csv.ts`）に切り出してテスト可能にした

## D4. CSV アップロードは対象外

backlog の項目は「テーブル選択 → CSV ダウンロード、CSV をドロップしてアップロード」だが、
**アップロードは実装しなかった**。

**根拠**: 物理ファイルへの取り込みには DDM（レコードレベルアクセス）が必要で、**未実装**である
（backlog「アップロード」＝別作業）。**無い機能の上に UI を作らない。**

## D5. タブのプレフィックス判定を 1 か所に集約する

`paneLabels.ts` に `PANE_PREFIXES` と `isPaneTab()` を置いた。

**根拠**: 判定が `App.vue` / `PaneTabs.vue` / `WorkspaceNode.vue` に散っており、
今回で 3 つ目のプレフィックスになった。**過去に実害が出ている**——`PaneTabs.isPane` に
`list:` を足し忘れ、タブを閉じるとセッション切断処理へ流れる不具合があった（コメントが残っている）。
種類が増えるたび各所を直して回る形は同じ再発を招く。

`WorkspaceNode.vue` だけは「どのペインを描くか」の個別判定が要るので共通化していない
（そこは**足し忘れると画面が出ない＝すぐ気づく**種類の失敗であり、静かに壊れない）。
