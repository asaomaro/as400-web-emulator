# 計画: SQL ペイン

## 分割の判定

**subtask にしない。** server 1 エンドポイント＋web-ui 1 ペイン＋配線 4 箇所で 1 PR に収まる。

## 順序

T1（共有切り出し）→ T2/T3（API）→ T5（CSV 純関数）→ T4（ペイン）→ T6（配線）
→ T7/T8（テスト）→ T9（ビルド）→ T10（実ブラウザ）→ T11（記録）

T1 を先にやるのは、T2 が `sourceSchema` を必要とするため。
T5 を T4 より先に置くのは、純関数を単体で固められるとペイン側が薄くなるため。
