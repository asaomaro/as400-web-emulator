# 要件: ホストサーバー機能を Web UI から操作する

## 背景 / 課題

backlog `hostserver.md` の「Web UI から操作（テーブル選択 → CSV ダウンロード、CSV をドロップして
アップロード）」。

前作業（`20260719-hostserver-mcp-tools`）で MCP には 10 本のツールを出したが、
**ブラウザから触れるのは依然としてジョブ／オブジェクト／ユーザー一覧だけ**である
（`HostListPane.vue` ＋ `/api/host/list/:kind` ＋ `/api/host/action`）。

SQL・スプール pull・IFS は core に実装済み、MCP からは叩けるが、**人間が画面から使えない**。
この PJ は ACS（IBM i Access Client Solutions）の代替を志向しており、ACS の
「データ転送」は GUI から表を選んで CSV に落とす機能である。そこに追いつく。

## 目的 / ゴール

**SQL の実行結果をブラウザで見て、CSV としてダウンロードできる**状態にする。

## スコープ

### 対象

- **SQL ペイン**（新規タブ種別）
  - SQL を入力して実行し、結果を表形式で表示する
  - 列名・行を CSV としてダウンロードする
  - エラー（SQLCODE / SQLSTATE）を人が読める形で表示する
- サーバー側の HTTP API（`/api/host/sql` 等）。MCP と同じ core 関数を叩く

### 対象外（この作業ではやらない）

- **CSV アップロード（表への取り込み）**。backlog の同項目に含まれているが、
  **物理ファイルへの書き込みには DDM（レコードレベルアクセス）が要り、それは未実装**である
  （backlog「アップロード」＝別作業 `20260719-hostserver-upload-ddm`）。
  **無い機能の上に UI を作らない。** アップロードはその作業のあとに別途載せる
- スプール pull / IFS の画面。SQL を先に出す（ACS データ転送の中心であり、
  1 PR の粒度としても適切）。必要なら後続で足す
- MSGW（前作業と同じ理由で未検証のため扱わない）

## 要検討（spec で決める）

### 1. 任意 SQL をブラウザから受け付けてよいか ★セキュリティ判断

前作業の D1 で「MCP には任意 CL を許す」と決めたが、**その根拠は MCP に限定されていた**——
「API トークンを持つ自動化向けであり、既に `run_steps` で同じ能力がある」。

**ブラウザには同じ理屈が通らない。** `/api/host/action` が任意 CL を拒んでいるのは、
まさにブラウザ（一般ユーザーを含む）が対象だからである。

ただし SQL の SELECT は CL 実行とは危険度が違う（`query` は SELECT 専用で、
INSERT/UPDATE/DDL は通らない）。**この差を根拠にできるかを spec で検討する**。
前作業の決定をそのまま持ってこない——対象読者が違う。

### 2. 結果セットの大きさ

前作業の review で挙げた既知の弱点——`query` は結果を全件取得してから返すため、
`maxRows` 相当は応答の切り詰めにしかならない。**同じ弱点をブラウザ経路にも持ち込むのか**、
それとも API 側で上限を強制するのかを決める。ブラウザは MCP より無邪気に
`SELECT * FROM 大きな表` を投げうる。

### 3. CSV の生成場所と文字コード

- サーバーで生成するか、ブラウザで生成するか
- Excel で開かれる前提なら BOM や改行コードの扱いを決める必要がある
- DBCS を含むデータの扱い（この PJ の中心的関心事）

### 4. 認可

AGENTS.md「アカウント・権限設計」の 3 層に当てはめる。
一般ユーザーが `srv:` を名指しできないこと（`assertProfileAccess`）は既存の仕組みに乗る。

## 受け入れ基準

- [ ] ブラウザで SQL を実行し、結果が表として表示される
- [ ] 結果を CSV でダウンロードできる
- [ ] SQL エラーが SQLCODE / SQLSTATE つきで表示される
- [ ] `docs/UI-DESIGN.md` の規約に従う（配色は CSS 変数・light/dark/system に追従）
- [ ] 認可がサーバー側で閉じている（UI の出し分けに依存しない）
- [ ] 結果セットの上限がサーバー側で強制される
- [ ] コンポーネントテストを追加。`npm run build -w @as400web/web-ui`（vue-tsc 込み）が通る
- [ ] **実ブラウザで確認**する（AGENTS.md の retro 反映——自動テストだけでは操作感を拾えない）
