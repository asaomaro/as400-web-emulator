## 実機検証（PUB400）

### 1. 自動サインオンあり（Web UI を実際に操作）

セッションを開いて情報ポップオーバーを開いただけ（**何も押していない**）:

```
種別      5250端末
ホスト     pub400.com
CCSID     939
画面      24x80
デバイス名   QPADEV000W               ← 実際に割り当てられた名前（設定は空＝ホスト採番）
ジョブ      372166/MARO/QPADEV000W   ← 番号・ユーザーまで解決できた
ラベル     session1
状態      接続中
```

- **取得ボタンは無い**（ポップオーバーのボタンは 0 個）
- **画面は `Display Messages` のまま**で、`DSPJOB` へ飛んでいない（画面テキストに DSPJOB の痕跡なし）

### 2. 資格情報なし（手サインオンの使い方）

WebSocket から `user` / `password` を渡さずにセッションを開いた場合:

```
opened: job = {"name":"QPADEV001W","system":"PUB400"}
jobinfo が来たか: false
```

- **装置名（＝ジョブ名）だけが出る**。ユーザー・番号は分からないので**照会しない**
  （他人のジョブを掴む危険を避ける。research F2）
- セッションは正常に開き、画面も出る

## 未検証・既知の制約

- **`zip-writer` の 4 件は環境要因**（このサンドボックスで `unzip` を実行できない `EACCES`）。本変更とは無関係
- **起動応答を送らないホスト**での実機確認はしていない（実機キャプチャ fixture での回帰テストのみ）。
  PUB400 は必ず送ってくる
- **社内機（SR-OSAKA）では未確認**。起動応答は RFC 4777 の標準機能で、当方は既に `IBMSENDCONFREC=YES` を
  申告しているが、実機で確かめてはいない
- **ジョブは接続時に 1 回だけ引く**。サインオンし直すなどでジョブが変わっても表示は更新されない
  （更新ボタンを置かないのが本件の主旨。装置名は変わらない）
- **照会は 1 セッションにつきコマンドサーバー接続を 1 本使う**（約 2 秒・使い終わったら閉じる）。
  接続を待たせてはいないが、同時に多数のセッションを開くと一時的に接続が増える
