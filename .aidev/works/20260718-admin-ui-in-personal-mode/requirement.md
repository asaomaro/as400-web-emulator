# 要件: 個人利用モードでも管理画面（セッション・ログ）を使えるようにする

## 背景 / 課題
個人利用（認証オフ）は「単一の信頼ユーザー」＝実質管理者として扱っているのに、
管理画面がまったく出ない。次の制約が重なっているため。

- `/api/admin/*` は `app.ts:41-45` の `if (deps.auth)` の中でしか登録されない（認証オフでは 404）。
- `requireAdmin()`（`auth.ts`）は `c.get("user")` が undefined だと 403。認証オフを通さない。
- 監査ログは `main.ts:127` の `installAuditBuffer` が `if (auth)` の中にあり、**認証オフでは収集されない**。
- UI の管理ナビは `authStore.isAdmin` でガードされており、認証オフでは出ない。

結果として、個人利用では**全セッションの一覧・切断ができず**、掴んだままのセッションを片付けられない。

## 目的 / ゴール
個人利用モードでも**セッション管理とログ**を管理画面から使えるようにする。

## スコープ
### 対象
- 認証オフでも `/api/admin/sessions` と `/api/admin/logs` を登録する。
- `requireAdmin()` を「**認証オフなら通す**」に変更する（`assertOwner` / `assertProfileAccess` と同じ
  「認証オフ＝全通過」の規則に揃える）。
- 監査ログを認証オフでも収集する（`installAuditBuffer` を常に実行）。
- UI: 認証オフでも「セッション」「ログ」を表示する。

### 対象外
- **ユーザー管理は非表示**（ユーザーが 1 人も存在しないため意味がない）。
  API も従来どおり認証時のみ登録する。
- 認証オンの挙動変更（admin は従来どおり 3 つとも使える／一般ユーザーは引き続き不可）。
- 管理画面の機能追加。

## 機能要件
- 認証オフ: 管理ナビに「セッション」「ログ」が出る。「ユーザー」は出ない。
- 認証オン・admin: 従来どおり 3 つとも出る。
- 認証オン・一般ユーザー: 従来どおり何も出ない（`/api/admin/*` は 403）。
- 認証オフでの監査ログが取得できる（収集されている）。

## 完了条件 (受け入れ基準)
- [ ] 認証オフで `/api/admin/sessions` が 200 を返し、全セッションが見える。
- [ ] 認証オフで `/api/admin/logs` が 200 を返し、イベントが記録されている。
- [ ] 認証オフでも `/api/admin/users` は登録されない（404）。
- [ ] 認証オン・一般ユーザーは `/api/admin/*` が 403（回帰）。
- [ ] 認証オン・admin は従来どおり 3 つとも 200（回帰）。
- [ ] UI が認証オフでセッション・ログのみ出す（ユーザーは出さない）。
- [ ] README の早見表「管理者画面」列が更新される。
