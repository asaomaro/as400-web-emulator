# 要件: PDF 作成・自動印刷の結果ステータス表示

## 背景 / 課題
自動出力（PDF 保存・物理印刷）は**失敗したときだけ**警告バーに出る。**成功したかどうかが画面から分からない**ため、
「本当に PDF ができたのか」「印刷に回ったのか」を確認できない（サーバーの出力先を直接見るしかない）。

`handleReport` は既に `{ pdfPath, printed }` を返しているが、`session-manager` が `void` で**捨てている**。

## 目的 / ゴール
受信スプールごとに、**PDF 作成の成否**と**自動印刷の成否**を画面で確認できるようにする。

## スコープ / 機能要件
- 受信スプールごとに次のステータスを表示する:
  - **PDF**: 保存成功（保存先が分かる）/ 失敗（理由）/ 設定なし / 自動出力オフでスキップ
  - **印刷**: 送信成功（プリンター名）/ 失敗（理由）/ 設定なし / 自動出力オフでスキップ
- スプール一覧の各行で**一目で分かる**こと（成功/失敗が区別できる）。詳細（保存先・理由）は選択時などで確認できること。
- 後から画面を開いても、そのセッションで受信済みのスプールのステータスが見えること。
- 既存の警告バー（失敗の通知）とサーバーログは維持する。

## 対象外 / 制約
- 出力設定の値の変更（別途プロファイル編集）。ここは**結果の表示のみ**。
- 過去セッション（切断後）の永続化はしない（セッション中のみ保持）。
- 信頼境界は不変（ブラウザから出力設定値は変更しない）。

## 完了条件 (受け入れ基準)
- [ ] PDF が作成できたスプールに成功ステータス（保存先が分かる）が表示される。
- [ ] 自動印刷が実施されたスプールに成功ステータス（プリンター名が分かる）が表示される。
- [ ] 失敗したものは失敗と分かり、理由が確認できる。
- [ ] 自動出力オフ中に受信したものは「スキップ」と分かる。
- [ ] 設定が無い場合は該当ステータスを出さない（または「設定なし」と分かる）。
- [ ] 後からタブを開いても既受信分のステータスが見える。

## 未確定事項 / 確認したいこと（spec で解消）
- **Q1（表示の粒度）**: 一覧行はアイコン等の簡易表示、詳細は選択スプールのヘッダに出す、で良いか。
- **Q2（失敗理由の保持）**: `handleReport` の warn 文言を結果にも載せて UI に出すか（現在は warn のみ）。
- **Q3（配送）**: 受信ごとに WS で push ＋ `printer-opened` で既存分を配送、で良いか。
