# 要件: 管理者画面（PR 2）

## 背景 / 目的
PR 1（認証・per-user 分離）の上に、管理者（role=admin）向けの管理機能を追加する。管理者は top 画面に
接続以外のセクションを持ち、クリックすると既存同様に**タブで展開**する。

## スコープ（対象）
- **ユーザー管理**: 一覧・作成・role/パスワード変更・削除、API トークン発行（1 回だけ表示）。users.json を更新。
- **セッション管理**: 全ユーザーの表示/プリンターセッション一覧（所有者付き）・任意セッションの切断。
- **ログ取得**: 直近の監査イベント（in-memory リングバッファ）を取得。
- **UI**: 管理者のみ top にセクション（ユーザー/セッション/ログ）。クリックで workspace のタブとして開く。
### 対象外
- 外部 IdP、監査ログの永続化（ファイル/DB）。細粒度権限。

## 非機能 / セキュリティ
- 全 admin エンドポイントは role=admin 必須（authMiddleware＋admin ガード）。パスワード/トークンは
  scrypt/sha256 のみ保存、発行トークンは 1 回だけ平文返却。users.json 書込は原子的に。
- 認証 OFF 時は管理機能を出さない（無認証運用は単一運用者前提）。

## 完了条件
- [ ] admin がユーザー CRUD＋トークン発行でき、users.json に反映される。
- [ ] admin が全セッション一覧・切断できる。非 admin は 403。
- [ ] admin が直近ログを取得できる。
- [ ] top の管理者セクションがタブで開き、上記を操作できる。
- [ ] 単体/統合テスト green、既存 green。
