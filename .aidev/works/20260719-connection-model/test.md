## 結合の確認

### 3 系統の命名整合（research A3）

`profile` / `connection` → `system` / `session` を MCP・WebSocket・REST で同時に置換した。
片方だけでは食い違う関係なので、**同一コミットで変えることが結合要件**だった。

| 系統 | 確認 |
|---|---|
| MCP | `mcp-config-tools.test.ts`（引数に `system`/`session` があり `profile`/`connection` が無い） |
| WebSocket | `ws-handler.test.ts`（`open` メッセージ） |
| REST | `config-routes.test.ts` / `host-lists.test.ts` |
| E2E スクリプト | `verify-mcp.mjs` / `verify-ws.mjs` を更新（実行は実機復旧待ち） |

### 信頼境界 5 層（親 spec の受け入れ基準）
