## 新しい不変条件の確認（`tab-visibility.test.ts`）

design の核心なので独立したテストを置いた。

| 確認 | 意図 |
|---|---|
| `visibleTabs` を呼んでも `group.tabs` が変わらない | **「隠す」が「閉じる」に化けない**——実体を触らなければ失われようがない |
| システムを切り替えるとタブが隠れ、戻すと現れる | セレクタは絞り込みであって破棄ではない |
| 所属が記録されていないタブは常に見える | 記録漏れで消えるより、余計に見えるほうが安全 |
| `closeSession` で `tabSystem` からも外れる | 閉じたタブの所属を残さない |
| `activeTabFor` が最後に見ていたタブへ戻す | 先頭固定より復帰が自然 |

## 未検証（正直に記す）

- **ブラウザでの実クリック確認は行っていない。** この環境で実ブラウザを操作できないため、
  検証は (a) コンポーネント単体テスト 307 件、(b) HTTP 経路の実サーバー確認、の 2 層に留まる。
  画面の見た目・操作感（カードの折り返し、その場編集の開き方、タブ帯の `＋`）は**目視未確認**
- プリンターセッションの実機確認は PUB400 の制約（特別権限 `*NONE`）で不可

## 途中で見つけて直した不具合

1. **`ConfigCard` の作成成功時にクラッシュ**（テスト追従で発覚）。
   `save()` が `editing = false` にしてから親がカードを消すため、その間に表示分岐が
   `props.system` を `undefined` のまま参照して落ちていた。新規作成時は `editing` を触らず、
   閉じるのを親に委ねる形へ修正
2. **ランチャーからの戻り道が無かった**（実装中に発覚）。ランチャーはワークスペースごと置き換える
   ため、タブ帯の `＋` に手が届かない。ヘッダーに「ワークスペースへ」を追加し、
   タブを選んだらランチャーを閉じるようにした。**モックでは気づけない種類の欠落**
3. **編集できないサーバー設定に編集ボタンが出ていた**。押しても 403 になるだけなので、
   `canEdit` で入口ごと塞いだ（認可の実体はサーバー側。ここは体験のため）
4. **`visibleTabs` が実配列を返していた**。呼び出し側が結果を書き換えると group が壊れる。
   コピーを返すようにして、不変条件を返り値の側でも守る
