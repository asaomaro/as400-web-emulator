# 要件: サーバー設定を一般ユーザーから隠す（admin 限定にする）

## 背景 / 課題

サーバー設定（`profiles.json`）は現在、**認証オンでも全ユーザーから見えて接続に使える**。
編集だけが `canEditProfiles`（認証オフ or admin）で守られており、参照と利用は素通しになっている。

- `GET /api/profiles`（`app.ts:71`）は利用者で絞り込まず常に全件返す。
  `includeSignon` で情報量は変わるが、**カード自体は一般ユーザーにも見える**。
- `profile` 名を指定した接続（WS / MCP）も役割を検査していない。

サーバー設定は「サーバーのパス書き込み・`lp` 実行・資格情報」に紐づく信頼設定の置き場であり、
一般ユーザーに露出させる前提ではなかった。一般ユーザーへ見せたい場合は、**明示的な公開設定を
別途追加する**方針が確認された（今回は追加しない）。

## 目的 / ゴール

**認証オンのとき、サーバー設定は admin だけが参照・利用できる。** 一般ユーザーからは存在しないものとして扱う。

## スコープ

### 対象
- `GET /api/profiles` を認可し、一般ユーザーには**空リスト**を返す。
- `profile` 名を指定した**接続・サインオン・プリンター出力解決**を認可し、
  一般ユーザーからの要求は **FORBIDDEN** にする（WS / MCP の全経路）。
- `ConnectionStore.assertOwner` に倣い、**ストア側で認可**する（呼び出し側の実装漏れを防ぐ）。
- ドキュメント（README / AGENTS）の可視範囲の記述を更新する。

### 対象外
- **公開フラグの追加**（一般ユーザーへ見せる手段）。別途検討。
- 認証オフの挙動変更（従来どおり全件参照・編集可）。
- admin の挙動変更（従来どおり全件参照・編集可）。
- 個人接続（`connections.json`）の可視範囲（現状維持）。

## 機能要件

- 認証オフ: 従来どおり全件見える・使える・編集できる（**変更なし**）。
- admin: 従来どおり全件見える・使える・編集できる（**変更なし**）。
- 一般ユーザー:
  - `GET /api/profiles` は `profiles: []` を返す（`editable: false`）。
  - `profile` 指定の接続（WS `open` / MCP `open_session` / `open_printer_session`）は FORBIDDEN。
  - `profile` 指定の `signon` ツールも FORBIDDEN。
  - プリンター出力設定（`resolvePrinterOutput`）も同様に拒否する。
- 認可は**単一の判定関数**に集約し、6 箇所の呼び出し元それぞれで独自判定しない。

## 非機能要件 / 制約

- 一覧から隠すだけの「security by obscurity」にしない。**参照と利用の両方**を塞ぐ。
- 既存の信頼境界（`canEditProfiles` / `.strict()`）は維持する。
- 永続データの形式は変更しない。

## 完了条件 (受け入れ基準)

- [ ] 認証オンの一般ユーザーに `GET /api/profiles` が空リストを返す。
- [ ] 一般ユーザーが `profile` 指定で接続しようとすると FORBIDDEN（WS / MCP の各経路）。
- [ ] 一般ユーザーの `signon`（profile 指定）が FORBIDDEN。
- [ ] admin と認証オフの挙動が従来どおり（回帰テストで担保）。
- [ ] 認可が単一関数に集約され、profiles を触る全経路が通っている。
- [ ] README / AGENTS の可視範囲の記述が更新されている。
