# 要件: 入力・ペースト時の検証と ACS 準拠メッセージ

## 背景 / 課題

現状、入力できない文字は**黙って捨てられる**。ACS は理由をメッセージで示し、
ペーストでは桁位置を保ったまま入力不可文字だけを反映しない。

とくに上書きペーストの実装が仕様と食い違っている。現状は文字ごと捨てるため
**後続が左へ詰まる**。数値項目 `123` に `3A5` を貼ると現状 `353`（期待 `325`）。

## 目的 / ゴール

打鍵・ペーストの検証と結果を ACS の挙動に合わせる。

## スコープ

### 対象

- 打鍵時のエラーメッセージ 4 種
- 上書きペーストの桁保持と欄またぎ
- 挿入ペーストの一括拒否
- 単一行コピーの矩形折返し

### 対象外

- コードページ固有の文字許容判定（送信時に core が行う既存責務）
- ペースト以外の経路（IME 合成・ドラッグ&ドロップ）

## 機能要件

### 1. 打鍵・削除時のメッセージ（ACS 原文）

| 状況 | メッセージ |
|---|---|
| 保護領域で文字入力・Backspace・Delete | `Cursor in protected area of display.` |
| 数値項目に許可外文字 | `Field requires numeric characters.` |
| 半角(A)項目に DBCS | `Field data must be alphanumeric.` |
| J 型(全角専用)項目に DBCS 以外 | `Double-byte character required as input.` |

数値項目の許可文字は **数字・`,`・`.`・`-`・`+`・空白**（現状の実装と同じ）。

**ペーストでは `Cursor in protected area of display.` を出さない**（保護欄は要件 4 の
走査で読み飛ばされるため、そもそもエラーにならない）。

### 1-b. クライアント側メッセージの寿命

- ACS はメッセージがクリアされるまで入力を受け付けないが、**この挙動は再現しない**
  （不便なため。ユーザー判断）。何か操作をすればクリアしてよい。
- **再現する挙動**: IBM i から受信したエラーメッセージが出ている状態で
  クライアント側メッセージを表示すると上書きされる。クライアント側メッセージが
  クリアされたら、**元の IBM i のメッセージへ戻る**。

### 2. 上書きモードのペースト

- **メッセージは出さない。**
- 入力不可文字は**その桁を消費し、元の文字を残す**（捨てて詰めない）。

  | 例 | 元の値 | 結果 |
  |---|---|---|
  | 数値に `1A` | 空白 | `1 ` |
  | 半角に `あ1いう2` | 空白 | ` 1  2` |
  | 数値 `123` に `3A5` | `123` | `325` |

  半角項目の DBCS は**半角スペース 1 桁**に置き換わる（1 ソース文字 = 1 桁）。
- J 型項目に DBCS 以外を貼った場合も同じ。**メッセージなし・その桁は元のまま・
  既に入力済みの DBCS にも影響を与えない**（全角 1 文字を半分だけ書き換えて壊さない）。
- 流し込みの規則は要件 5 に統一する。

### 3. 挿入モードのペースト

- 入力不可文字が 1 つでも含まれていたら**メッセージを出し、一切ペーストしない**。
  メッセージは打鍵時と同じ 4 種（項目種別に応じる）。
- **欄またぎは発生しない。** 収まらなければ `No room to insert data.`

### 4. 流し込みの規則（単一行・複数行で共通）

ペーストは**行ごと**に処理する。各行は次の順で流し込む。

1. 開始桁から**右へ**。開始桁が入力不可なら、**その行を右へ走査して最初の入力欄**から始める
2. その行で流し込む先が尽きたら、**次の行の同じ開始桁**へ回る（帯＝矩形を維持）
3. 複数行の場合、次のソース行は**前の行が使い終わった次の行**から始まる

**「縦か横か」は選択ではなく優先順位。右が先で、尽きたときだけ下。**

実例（すべてこの規則で説明できる）:

| 入力 | 配置 | 結果 |
|---|---|---|
| `5678` | `12` `34` が横並び | `56` `78`（右へ流れた） |
| `12345` | 行末から 2 桁目 | `12`/`34`/`5`（右が無く下へ） |
| `1111`/`2222`/`3333` | 幅 1・幅 4・幅 4・幅 4(2 桁右) | `1`/`111`/`2222`/`  3333` |

最後の例は、1 行目が幅 1 の行を使い切って次の行へあふれ、
2 行目はその次の行から、3 行目はさらに次の行の入力欄先頭から始まることを示す。

### 4-b. 単一行コピーの矩形折返し

複数行コピーの矩形ペースト（入力行が縦に連続する場合に下の行へ展開）は実装済み。
**単一行のコピーでも同じ折返しを行う。**

例: `12345` を行末から 2 桁目の位置で貼ると

```
12
34
5
```

## 非機能要件 / 制約

- 既存のガードテストを壊さない。仕様変更でテストが落ちる場合は
  削除せず新仕様へ書き換える（PJ 慣行）。
- メッセージは既存の `notice` 経路（`No room to insert data.` と同じ）で表示する。

## 完了条件 (受け入れ基準)

- [ ] 打鍵時、4 種の状況でそれぞれ ACS 原文のメッセージが出る
- [ ] 上書きペーストで、入力不可文字の桁が元の文字のまま残る（`123` + `3A5` → `325`）
- [ ] 上書きペーストが、保護項目を飛ばして次の入力欄へ割り込む
- [ ] 挿入ペーストは入力不可文字を含むと何も貼らず、該当メッセージを出す
- [ ] 挿入ペーストが収まらない場合は `No room to insert data.`
- [ ] 単一行コピーが矩形で下の行へ折り返す
- [ ] 各テストが修正前のコードで落ちることを確認済み

## 確定した論点

- **欄またぎ（横）と矩形折返し（縦）は競合しない。** `'12'` `'34'` は**同じ行に横並び**の配置であり、
  要件 2（欄またぎ）が働く。縦に連続する配置では要件 4（矩形折返し）が働く。
- **矩形折返しの幅は、カーソル桁から「その入力欄の右端」までの残り桁数。**
  画面行の右端ではない。
