# 要件: IFS ファイルの読み書き

## 背景 / 課題

backlog `hostserver.md` の「IFS ファイルの読み書き」。
PDF や CSV を IBM i 側に置く・取るための経路。原典で 4 ファイル 1,221 行と小さく、
費用対効果が高いと見積もっていた。

## 目的 / ゴール

ファイルサーバー（`0xE002` / 8473・9473）経由で IFS のファイルを読み書き・削除できる。

## スコープ

### 対象
- 接続・認証（既存 `server-connect` を再利用）
- ファイルの読み取り・書き込み・削除
- `packages/core` のライブラリとして実装

### 対象外
- ディレクトリ一覧・作成、属性変更、権限操作
- ストリーム API（大きなファイルの逐次処理）
- MCP・Web UI

## 完了条件 (受け入れ基準)

- [ ] ファイルを書き、読み戻して**内容が一致**する（実機）
- [ ] ファイルを削除できる（実機）
- [ ] 存在しないファイルの読み取りが原因の分かるエラーになる
- [ ] 要求組み立て・応答解析の単体テスト
- [ ] 資格情報が平文で出ない
