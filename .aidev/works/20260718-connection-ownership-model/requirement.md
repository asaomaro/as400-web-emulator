# 要件: 接続設定の所有モデルと種別の整理

## 背景 / 課題
現状の接続設定は「共有プロファイル（profiles.json）」と「個人接続（connections.json）」の 2 系統だが、UI の
挙動に不整合がある。
- **エミュレーター（表示）種別のプロファイル編集でも printer 出力設定が表示される**（プリンター専用の設定なのに）。
- 種別（エミュレーター/プリンター）が編集時に暗黙変更され得る（printer 設定の有無で種別が決まる鶏卵）。
- 認証オフ（単一利用）でも「共有/個人」の区別・ラベルが見えてしまい、単一利用者には不要な複雑さ。

## 目的 / ゴール
所有（共有/個人）と種別（エミュレーター/プリンター）を明確化し、利用環境に応じて過不足なく見せる。

## スコープ / 機能要件

### A. 種別（エミュレーター / プリンター）
- **新規作成時に種別を選ぶ**。作成後は**種別を変更できない**（エミュレーター⇄プリンターの切替不可）。
- **printer 出力設定（PDF 自動蓄積 / 自動印刷）は「プリンター種別」のときだけ表示・編集**する。
  エミュレーター種別の編集フォームには printer 設定を出さない。

### B. 所有（共有 / 個人）
- **認証オフ（マルチユーザー環境でない）**:
  - 作成する設定は**すべて共有**（profiles）にする。
  - **共有/個人の選択 UI を見せない**。一覧カードの**「共有」「個人」ラベルも見せない**。
- **admin ユーザー**:
  - 新規作成時に**共有 / 個人を選択**できる。
  - 既存設定の**共有 ⇄ 個人を変更**できる（profiles ⇄ connections の移動）。
- **一般ユーザー（認証オン・非 admin）**:
  - 従来どおり**個人接続のみ**作成・編集可。共有プロファイルは読み取り専用で接続に使える。

## 対象外 / 制約
- 認証・per-user 分離、信頼境界（printer/秘密は canEditProfiles 限定）の枠組みは維持。
- localStorage 廃止・サーバー保存は現状維持。

## 完了条件 (受け入れ基準)
- [ ] エミュレーター種別の編集で printer 出力設定が**表示されない**。プリンター種別のときだけ表示される。
- [ ] 編集フォームで種別（エミュレーター/プリンター）を変更できない（新規作成時のみ選択）。
- [ ] 認証オフでは、新規作成が共有扱いになり、共有/個人の選択 UI・一覧ラベルが**出ない**。
- [ ] admin は新規作成で共有/個人を選べ、既存設定の共有⇄個人を変更できる。
- [ ] 一般ユーザーは個人接続のみ（従来どおり）。信頼設定（printer/秘密）は admin/認証オフ限定のまま。

## 決定事項（requirement ゲート）
- **Q3（個人接続の種別・出力設定）**: 個人接続は**プリンター種別も可**（プリンターセッションを開ける）が、
  **PDF 出力設定（autoPdfDir/autoPrint）は付けられない**。出力設定＝信頼設定は**共有プロファイル限定**で
  信頼境界を維持する。

## 未確定事項 / 確認したいこと（design で解消）
- **Q1（新規“共有プロファイル”作成）**: 現状 UI では新規プロファイル（共有）を作れない（「＋新規接続」は個人のみ）。
  認証オフ/admin では**共有も UI から新規作成**できるようにする（＝profiles への add）。実装方針を design で固める。
- **Q2（共有⇄個人の移動）**: admin が既存設定の所有を切り替えるとき、profiles（name キー）と connections（id/owner キー）
  間の移動をどう扱うか（名前一意性・秘密の移送 passwordEnc/signon.passwordEnc・削除元/作成先）。design で確定。
- **Q4（既存データ・種別）**: 既存の profiles/connections はそのまま。種別は printer 設定の有無から確定（display 既定）。
  新規は明示選択、既存編集は種別固定。
