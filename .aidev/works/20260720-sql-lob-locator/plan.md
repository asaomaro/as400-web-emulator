# 計画: ロケーター経由の LOB 取得

## 分割の判定

**subtask にしない。** core の関数 1 つ＋配線で 1 PR に収まる。

## 順序

1. `retrieveLob`（core）＋分割受信 — 実機で早めに確かめる
2. `LobValue` への型変更（波及するので早めに）
3. `query` の `lob` オプション
4. **実機で 64KB 超の LOB**（分割受信の本番）
5. MCP / HTTP / Web UI / CSV
6. ロケーターの寿命を README に明記

## リスク

| リスク | 兆候 | 対応 |
|---|---|---|
| **分割受信が無限ループ** | 実機で止まらない | 進捗が無ければ打ち切る。上限バイト数でも縛る |
| 大きな LOB でメモリを掴む | 実機 | 既定 64KB・上限あり。**既定で取りに行かない** |
| CCSID の決め打ち | 文字化け | ホストの申告に従う。未知なら bytes のまま |
| 型変更の波及 | tsc | server / web-ui まで通す |
