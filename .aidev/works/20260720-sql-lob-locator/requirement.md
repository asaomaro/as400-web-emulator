# 要件: ロケーター経由で LOB の本体を取得する

## 背景 / 課題

利用者の要望:

> 本家がロケーターを使っているなら、こちらもロケーターを実装してください。

直前の作業（`20260720-sql-extended-format` / PR #100）で **LOB 列が結果セットに
含められる**ようになったが、返るのは**ロケーター（ハンドル）だけ**で、
値そのものは取得していない:

```json
{"ID":1,"C":{"kind":"lob","locator":256,"maxSize":0},"TXT":"plain"}
```

画面には `(LOB)` と出るだけで、**中身が見えない**。
「取得していないことが分かる」ところまでは達したが、**取得はできていない**。

## 原典の確認（AGENTS.md「既存プロトコル実装の移植」）

**推測せず直読した**（jtopenlite の `DatabaseConnection`）。

| 項目 | 実測 |
|---|---|
| 要求 ID | **`0x1816`** |
| ロケーターのハンドル | CP **`0x3818`**（4 バイト） |
| 要求サイズ | CP **`0x3819`**（4 バイト） |
| 開始オフセット | CP **`0x381A`**（4 バイト） |
| 圧縮指示 | CP `0x381B` |
| 変換指示 | CP `0x3805`（1 バイト） |
| 現在長を返す指示 | CP `0x3821`（1 バイト） |
| ORS ビットマップ | `SEND_REPLY_IMMED \| DATA_FORMAT \| RESULT_DATA` |

応答:

| CP | 内容 |
|---|---|
| **`0x3810`** | **LOB データ長**。先頭 2 バイトが長さの幅（0 / 4 / それ以外＝8 バイトで上下 32 ビット） |
| **`0x380F`** | **LOB データ本体**。`CCSID(2) + データ長(4) + 本体` |

`0x380F` は**バッファ単位で分割して届く**（原典は 64 KB のバッファで
`newLOBSegment` を繰り返し呼ぶ）。

## 目的 / ゴール

**LOB 列の値を実際に取得できる**ようにする。
画面で `(LOB)` としか見えなかったものを、中身が見える状態にする。

## スコープ

### 対象

- `retrieveLob(conn, locator, opts)` 相当の API（`packages/core`）
- 長さの取得（`0x3810`）と本体の取得（`0x380F`）
- **分割して届く本体の結合**
- 文字系 LOB（CLOB / DBCLOB）の**文字列への復号**（CCSID を見る）
- バイナリ（BLOB）は `Uint8Array` のまま返す

### 対象外

- LOB の**書き込み**（DDM 側も含む）
- ロケーターの明示的な解放（原典に該当があるか未確認。research で見る）
- 巨大 LOB のストリーミング API（まず「取れる」ことを優先。
  上限を設けて打ち切る形にする）

## 要検討（spec で決める）

### 1. いつ取得するか ★最重要

**`query()` が自動で本体まで取ると危険**である。前作業で実測したとおり、
LOB をインライン化すると 2 行で 8.4 MB になった。ロケーターを自動で辿れば
**同じことが起きる**（100 行 × 2MB = 200MB）。

候補:

- (a) 既定では取らず、**明示的に呼んだときだけ**取る（`retrieveLob`）
- (b) `query` にオプションを足し、上限バイト数まで自動で取る
- (c) 常に自動で取る（**採らない**——上記の理由）

**利用者にとって「画面で中身が見える」ことが目的**なので、
UI からどう呼ぶかまで含めて決める必要がある。

### 2. ロケーターの寿命

ロケーターは**いつまで有効か**。カーソルを閉じたら無効になるのか、
接続が生きていれば有効なのかを**実機で確かめる**。

`query()` は結果を返す前に `closeCursor` している。**もしカーソルに紐づくなら、
query が返したロケーターは既に無効**ということになり、設計が変わる。

### 3. 文字コード

`0x380F` は CCSID を伴って返る。DBCLOB は UTF-16（13488）の可能性が高いが
**推測しない**。実機で確かめる。

## 受け入れ基準

- [ ] ロケーターから LOB の**中身が取れる**（実機で往復確認）
- [ ] 文字系 LOB が正しく文字列になる（**実機で書いた値と一致すること**）
- [ ] 分割して届く本体が正しく結合される（バッファ長を超える LOB で確認）
- [ ] **既定で巨大なメモリを掴まない**（自動取得の是非は spec の判断に従う）
- [ ] ロケーターの寿命を実機で確かめ、**分かったことを記録する**
- [ ] 既存テストが緑 ／ `tsc -b` / lint / web-ui ビルド
- [ ] 画面で LOB の中身が見える（UI の扱いは spec で決める）
