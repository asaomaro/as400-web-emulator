# 仕様: ロケーター経由の LOB 取得

## D1. 同じクエリの中で取り切る（requirement の (a) は使えない）★

requirement は (a)「明示的に呼んだときだけ取る」を第一候補にしていたが、
**research F3/F4 の実測で否定された**——ロケーターは**接続に紐づく**（別接続では `-815`）。

私たちの接続は単発完結なので、画面で結果を見てからクリックした時点では
**接続は閉じており、ロケーターは無効**である。「クリックして展開」は成立しない。

**採る形**: `query()` にオプションを足し、**同じ接続の中で本体まで取る**。

```ts
query(conn, sql, { lob?: { maxBytes?: number } })
```

- **既定は取らない**（`lob` 未指定なら従来どおりロケーターのみ）。
  自動で取ると、前作業で実測した「2 行で 8.4MB」と同じ事故が起きる
- `lob.maxBytes` を超える LOB は**取りに行かず**、そうと分かる形で返す
- 取れたものは値（文字列 / バイト列）に置き換える

## D2. 取得結果の表し方

`LobPlaceholder` を拡張し、**取得できたか／なぜ取れなかったか**を保持する。

```ts
export interface LobValue {
  kind: "lob";
  locator: number;
  /** ホストが申告する全体のバイト長 */
  byteLength: number;
  /** 取得できた中身。未取得なら undefined */
  value?: string | Uint8Array;
  /** 未取得の理由。取得できたときは undefined */
  unavailable?: "not-requested" | "too-large";
}
```

**`value` を空文字で埋めない**。空の LOB と「取っていない」が区別できなくなる。
理由まで持つのは、画面に「大きすぎて省略」と出せるようにするため。

## D3. 上限の既定は 64 KB

原典が 64 KB のバッファで分割受信していることに合わせる。
「全部取る」を既定にしない——D1 の理由と同じ。

`maxBytes` は呼び出し側が上げられるが、**上限そのものは設ける**
（Web UI / MCP からは 1 MB を超える指定を受け付けない）。

## D4. 文字コードはホストの申告に従う

**実機で CCSID 273（EBCDIC）だった**。UTF-16 だと推測していたら外していた。

応答の `0x380F` が申告する CCSID でデコードする。**こちらで決め打ちしない。**
未知の CCSID なら `Uint8Array` のまま返す（壊れた文字列にしない）。

## D5. 分割受信に対応する

`0x380F` は 1 応答に収まらないことがある（原典はバッファ単位で繰り返す）。
`0x3810` が申告する全体長まで、**開始オフセット（`0x381A`）を進めて繰り返す**。

- 進まなくなったら打ち切る（**無限ループにしない**）
- 実機で 64 KB を超える LOB を作って確認する

## 2. API

```ts
/** ロケーターから LOB の本体を取る。分割受信を吸収する */
export async function retrieveLob(
  conn: DbConnection,
  locator: number,
  opts?: { maxBytes?: number; startOffset?: number }
): Promise<{ bytes: Uint8Array; ccsid: number; totalLength: number; truncated: boolean }>;
```

`query` はこれを使い、`lob` オプション指定時に各 LOB セルを埋める。

## 3. 上位への波及

- **MCP `host_sql`**: `lob` オプションを公開する（既定は取らない）
- **`/api/host/sql`**: 同上。上限は 1 MB
- **Web UI**: 「LOB の中身も取得する」チェックボックス。既定オフ。
  オンのとき `(LOB)` の代わりに中身を表示し、
  省略されたものは `(LOB: 大きすぎます)` と出す
- **CSV**: 取得できていれば中身、未取得なら `(LOB)`

## 4. 受け入れ基準

- [ ] ロケーターから中身が取れる（**実機で書いた値と一致**）
- [ ] **64 KB を超える LOB** で分割受信が正しく結合される（実機）
- [ ] 既定では取りに行かない（従来どおりロケーターのみ）
- [ ] 上限超過が `too-large` として区別できる
- [ ] CCSID をホストの申告どおりに扱う
- [ ] 画面で中身が見える／省略時はそうと分かる
- [ ] 既存テスト緑 ／ `tsc -b` / lint / web-ui ビルド
- [ ] **ロケーターが接続に紐づくこと**を README か JSDoc に明記する
      （知らないと「クリックで展開」を作ろうとして必ず嵌まる）
