# 要件: PDF 出力先を保存時に検証する

## 背景 / 課題

`autoPdfDir` は保存時に一切検証されず、**実際にスプールを受信するまで失敗が分からない**。

- `printer-output.ts:73` は `writeFile` を直接呼ぶだけで、サーバー側に `mkdir` も存在チェックも無い。
- 失敗は受信時に警告バー・ステータス（`PDF ✗`）として出るが、**設定を保存した場所と時点から離れている**。
- 実際に `/var/spool/as400-pdf` が存在せず `ENOENT` が発生し、原因特定に時間がかかった。

失敗が「直せる場所（設定画面）」で「直せるタイミング（保存時）」に出ないことが問題。

## 目的 / ゴール

**保存時に出力先を検証し、使えない場合はその場でエラーを返す。** ディレクトリの自動作成は行わない。

## スコープ

### 対象
- サーバー設定の保存（`POST /api/profiles` / `PUT /api/profiles/:name`）で `printer.autoPdfDir` を検証する。
  - 存在するか / ディレクトリか / **実際に書き込めるか**（一時ファイルの作成→削除で確認）
- 失敗時は **400** と、原因が分かるメッセージを返す。
- 検証に成功した場合、**解決後の絶対パス**を応答に含め、UI で確認できるようにする（相対パス・タイポの可視化）。
- UI: 保存失敗を設定フォームにインライン表示する。

### 対象外
- **ディレクトリの自動作成**（`mkdir -p`）。タイポでゴミディレクトリを掘るリスクを避けるため行わない。
- `autoPrint`（プリンター名）の検証。`lp` の宛先確認は環境依存が大きく、別途検討。
- 受信時の警告・ステータス表示の変更（**現状維持**。保存後に消える・権限が変わる可能性は残るため、
  実行時ハンドリングは引き続き必要）。
- 一般ユーザーへの開放（`autoPdfDir` は従来どおり認証オフ or admin のみ）。

## 機能要件

- `autoPdfDir` 未指定なら検証しない（従来どおり保存できる）。
- 指定時は次を検証し、いずれか満たさなければ 400:
  - パスが存在する（**存在しない場合は作成せずエラー**）
  - ディレクトリである（ファイルを指していたらエラー）
  - 書き込める（権限不足を検出する）
- エラーメッセージは原因を区別できること（存在しない / ディレクトリでない / 書き込めない）。
- 検証成功時、応答に解決後の絶対パスを含める。
- 検証は保存経路（`canEditProfiles` ゲートの下）でのみ行う。読み取り・接続時には行わない。

## 非機能要件 / 制約

- 検証のために**永続的な副作用を残さない**（一時ファイルは必ず削除する）。
- 信頼境界は不変。検証を追加するだけで、受理条件（admin or 認証オフ）は変えない。
- 既存の `profiles.json` は移行なしで読める（**起動時には検証しない**。既存設定を壊さない）。

## 完了条件 (受け入れ基準)

- [ ] ファイルを指したパスで保存すると 400（「ディレクトリでない」と分かる）。
- [ ] 書き込めないディレクトリで保存すると 400（「書き込めない」と分かる）。
- [ ] 正常なディレクトリなら保存でき、応答に解決後の絶対パスが含まれる。
- [ ] `autoPdfDir` 未指定の保存は従来どおり成功する。
- [ ] 検証で作った一時ファイルが残らない。
- [ ] 起動時（既存ファイル読み込み）には検証が走らない。
- [ ] UI で保存失敗の理由がフォーム上に表示される。
