# 計画: PDF 出力先を保存時に検証する

## 実装方針
純関数 `checkOutputDir` を先に作りテストで固める → 保存ルートに配線 → UI 表示 → ドキュメント。
受信時の挙動には触れない（保存時検証は実行時ハンドリングの置き換えではない）。

## 作業順序と依存関係
1. `output-dir.ts` に `checkOutputDir` を実装（依存: なし）
2. 単体テスト（存在しない/ファイル/書込不可/正常/一時ファイル残らない）（依存: 1）
3. `app.ts` の保存 2 ルートに配線・400 応答・`resolvedPdfDir`（依存: 1）
4. ルートのテスト（400 になり保存されない・成功時に絶対パス）（依存: 3）
5. UI: 保存失敗のインライン表示と解決パスの表示（依存: 3）
6. README 追記（依存: 3）
7. 全体検証（依存: 4,5,6）

## リスク / 留意点
- **root 実行だと書込不可テストが通らない**（root は権限を無視する）。テストでスキップ判定を入れる。
- 一時ファイルの後始末を `finally` で確実に行う（検証で副作用を残さない）。
- 保存前に検証する（不正設定を永続化しない）。

## テスト方針
- 単体: tmpdir に実ディレクトリを作って 5 パターン。
- 結合: `POST/PUT /api/profiles` が 400 を返し、`profiles.json` が更新されないこと。
