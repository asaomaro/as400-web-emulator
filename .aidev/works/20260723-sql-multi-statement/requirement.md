# 要件: SQL 画面で複数の文をまとめて実行し、結果をタブで見せる

## 背景 / 課題

SQL 画面は **1 回の実行につき 1 文**しか投げられない。

- 入力欄に `;` 区切りで複数の文を書いても、そのままサーバーへ送られて**構文エラー**になる
- 複数の表を見比べたいとき、文を書き換えて実行し直すしかない。**前の結果は消える**
- 結果領域は 1 つの表しか持たない（`columns` / `rows` / `resultSetId` が単一。`SqlPane.vue:37-40`）

ACS の SQL スクリプト実行や一般的な SQL クライアントは `;` 区切りの複数文を当たり前に扱う。
本アプリは「ACS でやっていたことを Web で」を狙っているので、ここが欠けていると素の SQL 作業に戻れない。

## 目的 / ゴール

`;` で区切った複数の文を**書いた順に実行**し、**結果セットごとにタブ**を出して見比べられるようにする。

## スコープ

### 対象

- 入力欄の SQL を `;` で分割して**順に実行**する
- 結果を返した文ごとに**結果領域のタブ**を作り、切り替えて表示する
- 各タブで既存の機能（列幅調整・CSV ダウンロード・続きの読み足し）が使える
- 文の途中で失敗したときの見せ方（どの文で・何が起きたか）
- 実行ログに**文ごと**の結果を残す

### 対象外

- SQL の整形・補完・シンタックスハイライト
- トランザクション制御（`COMMIT` / `ROLLBACK` の明示的な扱い）。既存の接続の挙動に従う
- 複数文の**並行**実行（順に実行する）
- タブの並べ替え・切り離し・固定

## 機能要件

1. 入力欄の内容を `;` で分割し、**空でない文だけ**を順に実行する
2. 分割は**文字列リテラル・引用符付き識別子・コメントの中の `;` を区切りと見なさない**
3. 結果セットを返した文は、**結果領域のタブ**として追加する（実行順に並ぶ）
4. タブの見出しから**どの文の結果か**が分かる（順番と文の要約）
5. 結果を返さない文（`INSERT` / `UPDATE` / `CREATE` 等）は**実行できない**（サーバーが SELECT 専用。research F2）。
   混ざっていたら**その文で止め、どの文が原因か**を示す（今は文全体が構文エラーになるだけ）
6. 途中で失敗したら、**どの文か**を示して止める（後続の文は実行しない）
7. 実行ログには**文ごとに 1 件**残す（既存のログ形式を踏襲する）
8. 単一の文を書いたときの見え方・操作は**今までと変わらない**（タブが 1 つだけ出る形でもよいが、邪魔にならないこと）

## 非機能要件 / 制約

- **接続とカーソルを持ちすぎない**。結果セットを開いたままにするとサーバー側の接続を掴む
  （`host-sql.ts` は結果セットに接続を結び付けて保持している）。同時に開く数の方針を決めること
- 実行のたびに前の結果を**確実に手放す**（既存の `releaseResultSet` と同じ規律）
- 文が 1 つのときに**余計な往復を増やさない**
- 既存の SQL 実行（単一文・ページング・CSV・ログ）に退行を出さない

## 完了条件 (受け入れ基準)

- [ ] `SELECT … ; SELECT … ;` を実行すると、結果タブが 2 つ出て切り替えられる
- [ ] 文字列の中の `;`（例 `WHERE NAME = 'A;B'`）で分割されない
- [ ] コメント（`--` / `/* */`）の中の `;` で分割されない
- [ ] 結果を返さない文が混ざったら、**どの文で止まったか**が分かる
- [ ] 途中で失敗したら、失敗した文が分かり、後続は実行されない
- [ ] 各タブで CSV ダウンロードと続きの読み足しができる
- [ ] 単一文のときの操作感が変わらない
- [ ] 実機（PUB400）で複数文の実行を確認できる

## 未確定事項 / 確認したいこと

- 結果を返さない文（DML/DDL）を**サーバーが実行できるか**。現状のコメントは「SELECT だけ」とある
- 結果セットを**同時にいくつ開けるか**（接続プールの上限との兼ね合い）。
  タブごとにカーソルを保持するのか、表示中のタブだけ保持するのか
- 分割の実装をどこに置くか（web-ui / server / core）。CSV や他の入口でも使うか
- 失敗時に**それまでの結果タブを残すか**（残す方が調査しやすい見込み）
