## 差し戻しと修正（2 ラウンド）

### ラウンド 1: test 工程での発見

**`/QSYS.LIB` で一覧のページングが無限ループした。**

全エントリの Restart ID が 0 で返るため、継続すると毎回先頭の数件が返る。
`--page` の 50 ページ上限で止めなければ終わらず、UI に載せればハングしていた。
`/home` では ID が正しく進むため、`/QSYS.LIB` 固有（原典の注記と鏡像で一致）。

`IfsListResult.canContinue` を足して塞ぎ、判定をユニットテストで固定、
実機で「`/QSYS.LIB` は止まる」「`/home` は 14 ページ完走する」の両方を確認した（decisions D6）。

### ラウンド 2: review 工程での発見（must 5 / should 6）

独立した第三者視点のレビューで、**自分では気づけなかった欠陥**が出た。詳細は `review.md`、
判断は decisions D7〜D9。特に重いもの:

- **テストが見せかけだった（M3）**。`canContinue` のテストが判定式をテスト側にコピーして
  そのコピーを検証しており、本体を壊しても緑のままだった。
  `IfsConnection.forTesting()` を足して連鎖ループ本体を偽接続で駆動する 18 件を追加。
  **ガードを `return true` に変異させると 3 件が落ちる**ことを確認済み（以前は 0 件）
- **エラーがすべて 502 に潰れていた（M2）**。spec の 404/403/409 を満たせない状態だった
- **終端フレームのエラーで接続を捨てていた（M4）**。`ifs-collect` の再帰走査が
  権限エラー 1 つで死ぬところだった
- **未使用エクスポート 9 個と、事実に反する根拠コメント（M5）**。
  「`ifs-list --raw` で使う」と書いたが実際の使用箇所は 0。grep で確認して削除した

## 未検証の範囲（deliver に引き継ぐ）

green だが**全数検証ではない**。以下は確かめられていない。

- **権限エラー（rc=13）の実機再現**。PUB400 は他ユーザーのホームも読める設定で、
  `/home/AWARREN` が 31 件返った。権限を拒否される状況を作れなかった。
  **【訂正】** ラウンド 1 でここに「コード上の対応のみで未検証」と書いたのは誤りだった——
  review（M2）で指摘されたとおり、当時は **rc=13 固有の処理そのものが無かった**（すべて
  `PROTOCOL_ERROR` に潰れていた）。現在は `ACCESS_DENIED` に投げ分け、偽接続の単体テストで固定してある。
  実機での再現だけが残る
- **解析失敗時の接続破棄**（D5）。偽サーバーのユニットテストでは確認したが、
  実機で解析が失敗する状況は作れていない
- **`/QSYS.LIB` の全件走査**。継続不可としたため、先頭 N 件までしか確認していない
- **4MB 超のファイル**。research で 4MB まで確認済みだが、それ以上は未確認
  （`MAX_FILE_BYTES` は 256MB）
