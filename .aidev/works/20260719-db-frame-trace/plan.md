# 計画: フレームトレース

## 分割の判定

**subtask にしない。** 純関数 1 つ＋適用 5 箇所で小さい。

## 順序

1. `frame-trace.ts`（純関数・単体テスト。マスクと切り詰めをここで固める）
2. `signon.ts` の private 実装を差し替え（**既存の挙動が変わらないこと**を確認）
3. 残り 4 箇所＋ server-connect に適用
4. README 追記
5. 実機でログを読む

**signon を先に差し替える**のは、既にトレースが動いている唯一の場所で、
**回帰の基準になる**から。ここが壊れなければ他への適用も安全。

## リスク

| リスク | 兆候 | 対応 |
|---|---|---|
| **マスク漏れ**（診断機能が漏洩経路になる） | 実機ログ | 単体テストで固め、**実機ログを目視**する |
| ログが溢れて実用にならない | 実機 | 値を 64 バイトで切る。切ったと明示する |
| signon の既存トレースの退行 | 既存テスト | 差し替え後に実機の signon ログを見る |
