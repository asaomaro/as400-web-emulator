# 要件: ホストサーバー接続にフレームトレースを入れる

## 背景 / 課題

backlog `hostserver.md`「SQL 実装からの積み残し」:

> **database 接続にフレームトレースを入れる**（signon にはある `traceFrame` 相当）
> - 資格情報の漏洩リスクは無い（そもそも出力しない）が、**障害時の診断手段が無い**。
>   実装中は使い捨てスクリプトを何度も書く羽目になった。**実利のある不足**
> - パスワード置換値（CP 0x1105）のマスクは signon と同じ扱いにする

**この指摘の正しさを、今回の 5 本で実体験した。** 直前の DDM 実装（作業 4）でも、
応答が期待と違うたびに使い捨てのスクリプトを書いて `console.log` でバイト列を
覗くことを繰り返した。トレースがあれば `LOG_LEVEL=debug` だけで済んだ。

## 実測: 穴は database だけではない

backlog は database だけを挙げているが、**実際は 4 接続すべてに無い**:

| 接続 | フレームトレース |
|---|---|
| `signon.ts` | ✅ ある（`traceFrame`。パスワードをマスク） |
| `db/db-connection.ts` | ❌ 無い |
| `command/command-connection.ts` | ❌ 無い |
| `spool/netprint-connection.ts` | ❌ 無い |
| `ifs/ifs-connection.ts` | ❌ 無い |
| `server-connect.ts`（共通の認証） | ❌ 無い |

これらは**同じ 20 バイトヘッダー＋LL/CP の形式**を共有しているので、
**1 つの共通ヘルパで 5 箇所すべてに効く**。database だけに入れるのは、
同じ作業を将来 4 回繰り返すことになる。

> 作業 4 で実装した DDM は**フレーム形式が違う**（2 バイト長・6 バイトヘッダー）ので、
> 同じヘルパは効かない。扱うかどうかは spec で決める。

## 目的 / ゴール

**`LOG_LEVEL=debug` だけで、ホストサーバーとのやり取りをバイト単位で追える**ようにする。
使い捨てスクリプトを書かずに障害を切り分けられる状態にする。

## スコープ

### 対象

- 共通のフレームトレースを 1 箇所に切り出す（`signon.ts` の private 実装を移す）
- 20 バイトヘッダー形式の 4 接続 ＋ `server-connect.ts` に適用する
- **秘密のマスク**（下記）

### 対象外

- トレースの永続化・リプレイ（5250 には `trace/` があるが、別の話）
- 出力形式の設定可能化（まず出ることを優先する）

## 要検討（spec で決める）

### 1. マスクする対象 ★セキュリティ

backlog は「パスワード置換値（CP 0x1105）」を挙げる。**それだけで十分かを確かめること。**

- `signon.ts` は `CP.password` をマスクしている
- `server-connect.ts` も同じ CP でパスワード置換値を送っている
- **他に秘密を載せる CP が無いか**を実際に調べる（推測しない）

AGENTS.md「セキュリティ」——ログにも値を出さない。
**マスク漏れは、診断のために入れた機能が漏洩経路になる**という最悪の形になる。

### 2. DDM を含めるか

フレーム形式が違うため別実装が要る。**入れる価値と、作業 4 の直後に触るリスク**を
天秤にかける。入れないなら backlog に残す。

### 3. 出力量

SQL の結果セットは大きい。全フレームを hex で出すと**ログが溢れる**。
signon の実装は LL/CP ごとに区切って出しているので、それに倣うか、長い値を切るかを決める。
**切るなら「切った」ことが分かるようにする**（黙って切らない）。

## 受け入れ基準

- [ ] `LOG_LEVEL=debug` で 4 接続すべてのフレームが追える
- [ ] **秘密がマスクされている**ことを、実機のログを実際に見て確認する
- [ ] `LOG_LEVEL` が debug でないときは**整形コストが発生しない**
      （作業 3 で `isDebugEnabled()` を用意した。これを使う）
- [ ] 単体テスト（マスク・長い値の扱い）
- [ ] `tsc -b` / lint / 既存テスト緑
- [ ] **実機で実際に出力を見る**（出力があることだけでなく、読んで役に立つか）
