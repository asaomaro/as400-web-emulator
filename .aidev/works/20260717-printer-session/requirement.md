# 要件: TN5250E プリンターセッション（スプール出力をブラウザで帳票表示）

## 背景 / 課題
現状の Web エミュレーターは 5250 **表示**セッションのみをサポートする。IBM i の帳票（スプールファイル）
を手元で確認するには、別途 5250 表示画面から `WRKSPLF` を辿るか、ACS の 5250 プリンターエミュレーション
に頼るしかない。ACS 相当の操作体験を目指すうえで、**プリンターセッションでスプール出力を直接受信して
ブラウザで閲覧できる**手段が欠けている。

5250 プリンターセッションは表示セッションとは別物で、
- 交渉が別（TN5250E＝RFC 2877/4777、端末タイプはプリンター機種、`DEVNAME` 指定）
- データストリームが別（**SCS＝SNA Character String**。画面オーダーではなく印刷制御コード列）
- 出力先が別（画面バッファではなく「帳票」）
という独立サブシステムの追加になる。

## 目的 / ゴール
ホストのスプールファイルを 5250 プリンターセッションとして受信し、**改ページ・行構成を保った整形帳票
（PDF 相当）としてブラウザに表示（必要に応じ保存）できる**状態にする。SBCS（英数）と
DBCS（日本語・CCSID 1399）の双方で文字化けなく帳票化できることをゴールとする。

## スコープ
### 対象
- TN5250E でのプリンターデバイスとしての接続・交渉（`DEVNAME` 指定、プリンター端末タイプの申告）。
- ホストが送る **SCS データストリームの受信とデコード**（改行・改ページ・行間・桁位置など帳票整形に
  必要な範囲の制御コード）。
- 受信データの **帳票としての整形表示（PDF 相当・改ページ保持）**、およびブラウザからの保存。
- **印刷完了のホストへの応答**（スプールが正常終了扱いになること）。
- **SBCS ＋ DBCS（CCSID 1399・日本語）** の帳票化。
- 表示セッションとは独立したプリンターセッションとして扱えること（同時併存を妨げない）。

### 対象外
- グラフィックス・バーコード・IPDS（3812 の SCS の範囲を超える高機能印刷）。
- 実物プリンターへの物理出力（あくまでブラウザ内の閲覧・保存）。
- 用紙トレイ・両面・ステープル等のフィニッシング制御の完全再現。
- Host Print Transform 前提の PCL/AFP 等ホスト側変換出力への対応（要否は research/spec で判断）。
- スプールファイルの選択・管理 UI（キュー閲覧・保留/解放操作などの帳票管理機能）。

## 機能要件
- プリンターデバイスとして IBM i（既定 PUB400）へ接続し、任意の `DEVNAME` を申告できる。
- 接続中、そのデバイスの出力キューに回されたスプールファイルを受信できる。
- 受信した SCS を帳票へ整形し、改ページを保った形でブラウザに表示できる。
- SBCS と DBCS（日本語）の文字を正しくデコードして表示できる。
- 受信・整形した帳票をブラウザから保存できる。
- 受信完了時にホストへ印刷完了応答を返し、ホスト側でスプールが正常終了する。

## 非機能要件 / 制約
- 既存の表示セッション実装（`Session5250` 等）を壊さない。プリンターは独立系統として追加する。
- DBCS 帳票は CCSID 1399 セッションを前提とする（表示 DBCS と同じ制約）。
- 実機検証は既定 PUB400 を用いる（`scripts/verify-*` の実行規約に従う）。秘密情報はコミットしない。
- 参照実装として tn5250 の `lp5250d`（プリンター実装）を活用できる（表示 DBCS と違い実装が存在する）。

## 完了条件 (受け入れ基準)
- [ ] TN5250E でプリンターデバイスとして接続でき、指定 `DEVNAME` の仮想プリンターデバイスがホスト側に
      生成される（**または実機で生成不能と判明し、その事実と代替が記録される**）。
- [ ] そのデバイスの出力キューへ回したスプールファイルを SCS として受信できる。
- [ ] 受信 SCS を改ページを保った整形帳票としてブラウザに表示できる。
- [ ] SBCS と DBCS（CCSID 1399・日本語）の双方で文字化けなく帳票化できる。
- [ ] 受信完了をホストへ応答し、対象スプールがホスト側で正常終了扱いになる。
- [ ] 整形帳票をブラウザから保存できる。

## 未確定事項 / 確認したいこと（research/spec で解消）
- **PUB400 での実現性（最重要・未検証）**: 一般ユーザーが TN5250E プリンター接続で仮想プリンター
      デバイスを自動生成でき（`QAUTOVRT` 等）、自前のスプールをそのデバイスの出力キューへ回せるか。
      ここが不可なら PUB400 では受け入れ基準の実機検証ができず、スコープ・検証手段の見直しが要る。
- **SCS の必要範囲**: 帳票整形に最低限必要な SCS 制御コード群（改行・改ページ・行間・桁移動・
      透過・DBCS SO/SI 相当）の実挙動。tn5250 `lp5250d` と実機採取で確定する。
- **出力形式の実現手段**: 「PDF 相当の整形帳票」をブラウザでどう出すか（クライアント生成か、
      サーバ生成か、等幅グリッドの HTML 表現で足りるか）は spec の設計判断。要件は「改ページを保った
      整形帳票として閲覧・保存できる」ことまで。
- **Host Print Transform の要否**: 標準は SCS 受信だが、ホスト設定次第で変換済みデータ経路が
      あり得る。対象に含めるか research/spec で判断する。
