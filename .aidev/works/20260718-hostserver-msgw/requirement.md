# 要件: MSGW（メッセージ待ち）の検出と応答

## 背景 / 課題

backlog `hostserver.md` の「MSGW の検出」。

以前のセッションで、プリンターが `MSGW` で止まる問題を調べ、**PUB400 では `CHGWTR` が
権限不足で不可**と確定していた。backlog には「**検出だけなら可能かもしれない・要実機確認**」と
条件付きで積んである。

前作業でネットワーク印刷サーバー（`0xE003`）を実装した際、
`RETRIEVE_MESSAGE`(0x0011) / `ANSWER_MESSAGE`(0x0012) という **`CHGWTR` とは別経路**が
使えることが分かった。権限の要件が違う可能性があるため、実機で確かめる。

## 目的 / ゴール

スプールに付いた応答待ちメッセージを**取得**し、必要なら**応答**できるようにする。

## スコープ

### 対象

- `RETRIEVE_MESSAGE` によるメッセージ取得（ID・本文・ヘルプ・応答ハンドル）
- `ANSWER_MESSAGE` による応答
- 一覧の状態（`MESSAGE_WAIT`）との連携

### 対象外

- writer の開始・終了（`STRPRTWTR` / `ENDWTR` 相当）
- メッセージ待ちの監視ループ・通知（利用側の責務）
- MCP・Web UI
