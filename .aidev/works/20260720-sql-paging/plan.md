# 計画: SQL のページング

## 分割の判定

**subtask にしない。** サーバー 1 モジュール＋UI 1 ペインで 1 PR に収まる。

## 順序

1. `result-set-store.ts`（セッション保持・タイムアウト・上限）＋単体テスト
2. `/api/host/sql` の拡張と `/next`
3. **実機で stream の読み進み**（1000 件超・ブロック境界）
4. UI（ドロップダウン・読み足し・状態表示）
5. 実ブラウザ（キーボード／スクロール）

**1 を先に**やるのは、タイムアウトと上限を注入可能にしておかないと
テストが実時間に依存するため。

## リスク

| リスク | 兆候 | 対応 |
|---|---|---|
| **接続が残る**（掴みっぱなし） | 実機・プロセス終了時 | アイドル 60 秒＋上限 4＋終了時クローズ。テストで固定 |
| stream がブロック境界で壊れる | 実機 | 1000 件超で確認。#99 と同じ轍を踏まない |
| 二重読み足し | 実ブラウザ | `loadingMore` で塞ぎ、実ブラウザで確認 |
| 期限切れが黙って空表示になる | 実ブラウザ | 404 を明示メッセージに |
| MCP への波及 | server テスト | `maxRows` 経路を変えない。回帰で確認 |
