# 仕様: SQL 結果の遅延ローディング

## D1. Web UI 用に結果セットセッションを持つ（MCP は変えない）★

research F3/F4 の実測により、**単発完結のままページングする選択肢は無い**
（1 ページごとに接続を張り直すと 6 秒/ページ）。

**採る形**: `/api/host/sql` が結果セットのハンドルを返し、
続きは `/api/host/sql/:id/next` で取る。サーバーは接続とカーソルを保持する。

**`20260719-hostserver-mcp-tools` の D2（単発完結）を覆すものではない。**
MCP の `host_sql` は**現状どおり単発完結のまま**変えない——
あちらは対話的に単発で叩かれる用途で、ページングの動機が無い。
状態を持つのは**画面の都合**なので、画面の経路にだけ持たせる。

### 安全弁（状態を持つ以上、必須）

| 項目 | 値 | 理由 |
|---|---|---|
| アイドルタイムアウト | **60 秒** | 接続を掴み続けない。スクロール間隔としては十分 |
| 同時に開ける結果セット | **1 ユーザー 4 つ**まで | 溜め込ませない。超えたら**最も古いものを閉じる** |
| 所有 | 作成した利用者のみ | 既存の `assertOwner` に乗せる（認証オフは単一利用者） |

**サーバー終了時に必ず閉じる。** 掴んだ接続を残さない。

## D2. `stream()` を使ってカーソルを保持する

`query()` はカーソルを閉じてしまうので使えない。既存の `stream()`
（`AsyncGenerator`）を保持し、必要な件数だけ `next()` する。

```ts
interface ResultSetSession {
  id: string;
  owner?: string;
  conn: DbConnection;
  rows: AsyncGenerator<Row>;
  columns: ColumnMeta[];
  done: boolean;
  lastUsed: number;
}
```

> `stream` は「ブロッキング係数を跨ぐ規模が未検証」と backlog にあったが、
> `20260719-hostserver-mcp-tools` で **`query` 側は 250 行で確認済み**。
> `stream` 自体は本作業が初めての実利用になるので、**実機で確かめる**。

## D3. 「1 度に取得する件数」の意味に変える

- ドロップダウン: **50 / 200 / 500 / 1000**（既定 200）
- 上限としてではなく、**1 回の読み足しで取る件数**として使う
- サーバー側の受け付け上限は 1000 のまま（1 回の応答が過大にならないように）

## D4. 「続きがあるか」は 1 件多く読んで判断する

`stream` から `n+1` 件読み、**n 件返して 1 件を次回の先頭に持ち越す**。
`n` 件しか読めなければ `hasMore: false`。

**SQLCODE 100 を待つ形にしない**——「最後のブロックがちょうど n 件」のときに
余計な往復が発生し、しかも #99 で踏んだ「ちょうど倍数」の経路に近くなる。

## D5. 読み足しの起点は 3 つとも同じ経路

**End / PageDown / スクロール**のいずれも同じ関数を呼ぶ。

- スクロール: 表の外枠（`.rows-scroll`）の `scroll` を見て、
  **下端から 200px 以内**に来たら読み足す
- キーボード: ペインに focus がある状態で `End` / `PageDown`
- **読み足し中は二重に走らせない**（`loadingMore` で塞ぐ）

## D6. CSV は「今表示している行」であることを明示する

現状も表示中のものを出しているが、ページングで**「どこまで読んだか」で内容が変わる**。
ボタンの表示を **「CSV をダウンロード（表示中の N 件）」** にして、
利用者が何を得るのか分かるようにする。

> 「全件 CSV」は別の機能（サーバー側で全件を回して流す）であり、本作業では扱わない。
> **黙って一部だけ出すことをしない**のが要点。

## 2. API

```
POST /api/host/sql
  body: { source, sql, pageSize?, lobMaxBytes? }
  200:  { resultSetId, columns, rows, hasMore }

POST /api/host/sql/:id/next
  body: { pageSize? }
  200:  { rows, hasMore }
  404:  期限切れ／存在しない（画面は「セッションが切れました。再実行してください」）

DELETE /api/host/sql/:id     … 画面を閉じたときの後始末（任意）
```

`maxRows` は**互換のため受け付ける**（指定時は従来どおり全体の上限として振る舞い、
`resultSetId` を返さない）。MCP はこちらを使い続ける。
