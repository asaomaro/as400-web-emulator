# 仕様: 画面全体の自由カーソル（非入力エリアへのカーソル配置）

## 概要

Web エミュレーターのカーソルを「入力欄に紐づく実装」から「**画面全体を自由に動く単一カーソル**」へ変える。
論理カーソル位置 `(row, col)` を単一の真実（source of truth）とし、表示オーバーレイ・AID 送信・入力欄フォーカスを
そこから導出する。入力欄セルではキャレット＝カーソル、非入力セルではオーバーレイ＝カーソル、を一貫させる。
per-char span 化はせず、固定等幅グリッドの座標計算で実現する。

## 設計方針

- **単一の論理カーソル**を持つ（composable `useCursor` に集約、または EmulatorPane に一元化）。初期値はホストの
  `snapshot.cursor`。ユーザーのクリック/矢印で上書きし、次のホスト画面（snapshot 更新）でリセット（＝ホスト
  カーソルに戻る。既存の `cursorOverride` の考え方を踏襲・拡張）。
- **描画**: カーソルオーバーレイは「有効カーソル（override ?? snapshot.cursor）」に追従（現状 `snapshot.cursor`
  固定を是正）。有効カーソルが入力欄セルにあり、その `<input>` にフォーカスがある場合は native キャレットを正とし
  オーバーレイは隠す。それ以外はオーバーレイを表示。
- **入力欄との協調**は既存の「native caret を単一の真実に据える」設計（AGENTS.md）を踏襲し、論理カーソルと
  native caret を相互に同期する。
- **座標変換の精度**: クリックの pixel→セルは **実測字幅**（描画済みセル 1 個の `getBoundingClientRect().width`、
  または不可視の測定要素）で算出する（現状 `fontPx*0.6` 近似の 1 桁ズレを解消）。行高は `1.25em` 実測。

## 対象範囲

- `packages/web-ui/src/components/ScreenGrid.vue`: オーバーレイ位置を有効カーソルに、`onGridClick` の座標精度、
  クリック/矢印時のフォーカス・キャレット反映。
- `packages/web-ui/src/components/EmulatorPane.vue`: 論理カーソル状態、矢印キーのセル単位移動、フォーカス/blur の
  調停、AID 送信（既存）。
- （任意）`packages/web-ui/src/composables/useCursor.ts`: カーソル状態と移動ロジックを純関数化（テスト容易化）。
- 非対象: `packages/core` / `packages/server`（プロトコル・MCP・WS は不変。カーソルは AID 送信で既に扱える）。

## インターフェース / データ構造

- 論理カーソル: `cursor: { row: number; col: number }`（1 始まり）。`effectiveCursor = override ?? snapshot.cursor`。
- `ScreenGrid` props 追加: `cursor: { row; col }`（有効カーソル）。オーバーレイはこれを使う。
- 移動純関数（`useCursor` 例）:
  - `moveCursor(cur, dir, rows, cols): {row,col}` — dir=up/down/left/right。左右は行送り/戻し、上下は同桁で行移動。
    画面端はクランプ（またはラップ。§振る舞いで確定）。
- フィールド判定: `fieldAt(row, col, fields): Field | undefined`（既存の描画側マップを再利用/共有）。

## 振る舞いの詳細

### クリック
- `onGridClick`: 実測字幅で `(row,col)` を算出 → 論理カーソルを設定。
  - そのセルが**編集可能フィールド内**なら該当 `<input>` を focus し、キャレットを `(col - field.col)` に。
  - **保護フィールド/非入力**なら、フォーカス中の入力欄を blur し、オーバーレイを当該セルに表示。
- 入力欄を直接クリックした場合は従来どおり（native focus＋クリック桁）。論理カーソルもその位置に同期。

### 矢印キー（セル単位・全画面）
- 論理カーソルを 1 セル移動し、フォーカスを調停する:
  - 新セルが**編集可能フィールド内** → その `<input>` を focus、キャレットを桁に合わせる。
  - 新セルが**非編集** → 入力欄を blur、オーバーレイ移動のみ（そのセルでの文字入力はしない）。
- **入力欄内でのキャレット移動との整合**: フォーカスが入力欄にあり、Left/Right でフィールド内にキャレットが
  収まる間は従来どおり欄内移動（onInputKeydown）。**フィールド端でさらに外へ出る Left/Right**、および Up/Down は
  論理カーソルのセル移動へ委譲する（＝欄から抜けて隣接セルへ）。
- 端の扱い: 左端で Left は前行末尾へ、右端で Right は次行先頭へ（行送り）。最上行 Up / 最下行 Down は現状維持
  （クランプ）。※ホスト送信はしない（ローカル移動のみ）。

### AID 送信
- Enter / F キー等は**有効カーソル位置**を伴って送る（既存の `cursor.value` 経路を論理カーソルに統一）。
  例: F4 を非入力ラベル上で押すとその位置でプロンプトが開く。

### ホスト画面更新
- 新しい snapshot 受信で override をクリアし、ホストの `snapshot.cursor` に戻る（既存 watch を踏襲）。

## エラー処理 / 異常系

- 算出座標が画面範囲外はクランプ（設定しない）。
- フィールドが 0 個の画面でも矢印でオーバーレイが全画面移動できる（入力欄不在でも破綻しない）。

## ドメイン固有の考慮

- 通信中（busy）は入力プロテクト中のためカーソル移動も抑止（既存プロテクトに合わせる）。
- DBCS セル（lead/tail 2 桁）上のカーソルは lead 桁に丸める（2 桁 1 文字の整合）。

## 受け入れ基準との対応

| 完了条件 | 実現方法 |
|---|---|
| 1. クリックで任意セルにカーソル＋表示 | `onGridClick` 精度化＋論理カーソル設定＋オーバーレイ追従＋focus/blur 調停 |
| 2. 矢印でセル単位の全画面移動 | `moveCursor` ＋フォーカス調停（入力欄は編集、非入力は表示のみ） |
| 3. AID にカーソル反映 | AID 送信を有効カーソルに統一（既存経路） |
| 4. 非回帰 | 欄内編集・Tab・ホイール・遷移後フォーカスは既存動作を保持（矢印の欄内挙動は端でのみ委譲） |
| 5. span 化しない | 座標計算＋単一カーソルモデル。`v-memo`/run まとめは維持 |

## 残課題 / design への申し送り

- **カーソルの状態遷移**（入力欄内キャレット ⇄ 非入力オーバーレイ、クリック/矢印/ホスト更新）を状態図で固める。
- native caret と論理カーソルの同期タイミング（focus/blur/keydown/snapshot 更新）をシーケンスで明確化。
- Tab（フィールド間ジャンプ）と矢印（セル単位）の役割分担の最終確定。
