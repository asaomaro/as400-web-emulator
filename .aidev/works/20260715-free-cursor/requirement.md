# 要件: 画面全体の自由カーソル（非入力エリアへのカーソル配置）

## 背景 / 目的

- 現状、Web エミュレーターのカーソルは入力フィールドに紐づいており、**非入力（表示専用/保護）エリアへ
  カーソルを置けない**。矢印キーはフィールド間ジャンプのみ、クリックで非入力エリアを指してもカーソル
  オーバーレイが追従しない（`onGridClick` は座標→セルを算出し emit するが、オーバーレイは `snapshot.cursor`
  を見ているため見た目が動かない）。
- ACS / 実機 5250 は「**画面全体を自由に動く単一カーソル**」で、非入力エリアにもカーソルを置ける。
  カーソル位置は AID 送信（F4=プロンプト、Help 等）に使われるため、任意位置に置けることは操作上重要。
- これを実現し、ACS 同等の操作感にする。

## 完了条件（受け入れ基準）

1. **クリック**: 画面上の任意セル（入力欄・非入力エリアいずれも）をクリックすると、カーソルがその位置へ移動し、
   カーソルオーバーレイがその位置に表示される。入力欄なら該当欄にフォーカスして当該桁にキャレット、
   非入力なら欄の外でオーバーレイ表示のみ。
2. **矢印キー**: カーソルがセル単位で全画面を移動できる（フィールド間ジャンプではなく）。入力欄セルへ入ったら
   編集可能、非入力セルではオーバーレイ表示のみ（そのセルでの文字入力はしない）。行末/画面端の挙動を定義する。
3. **AID 反映**: カーソル位置が AID 送信（Enter / F キー等）に正しく反映される（例: F4 プロンプトが正しい位置で開く）。
4. **非回帰**: 既存のフィールド編集（上書き/挿入・欄内任意桁入力）・Tab 移動・マウスホイールのページ送り・
   画面遷移後の自動フォーカスが回帰しない。
5. **実装方針**: 非入力エリアを 1 文字幅 span で埋める方式は採らず、**固定等幅グリッドの座標計算＋単一カーソル
   モデル**で実現する（描画性能・`v-memo` 設計を維持）。

## スコープ

- 対象: `packages/web-ui`（`ScreenGrid` / `EmulatorPane` / カーソル状態モデル、クリック・キー処理）。
- 非対象: `packages/core` / `packages/server` のプロトコル・MCP・WS メッセージ（カーソルは既に AID 送信で扱える）。

## 制約 / 留意

- クリックの座標→セル変換は**実測字幅**で精度を確保する（現状 `fontPx*0.6` 近似で 1 桁ズレの恐れ）。
- **native input のキャレットと自由カーソルを単一のカーソルモデルに統合**する（retro の課題「カーソル/編集
  モデルの一元化」）。入力欄セルではキャレット＝カーソル、非入力セルではオーバーレイ＝カーソル、を一貫させる。
- 保護フィールドはカーソルを置けるが編集不可（現状どおり）。
