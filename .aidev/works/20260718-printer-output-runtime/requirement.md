# 要件: プリンター出力エラーの可視化と実行時 ON/OFF

## 背景 / 課題
PDF 自動蓄積・自動印刷が失敗しても、**サーバーログにしか出ず画面から気づけない**。実際に次のエラーが出ていた:

```
{"level":40,"component":"printer-output",
 "msg":"PDF 保存に失敗（/var/spool/as400-pdf）: ENOENT: no such file or directory, open '/var/spool/as400-pdf/…pdf'"}
```

また、出力設定はプロファイルで固定されており、**セッション実行中に一時的に止められない**（出力先が壊れている・
今は印刷したくない、等の状況で困る）。

## 目的 / ゴール
- プリンター出力（PDF 保存 / 自動印刷）の**失敗を画面上で確認できる**ようにする。
- プリンターセッション画面から**自動 PDF / 自動印刷の有効・無効を実行時に切り替えられる**ようにする。

## スコープ / 機能要件

### A. エラーの可視化
- 出力（PDF 保存・`lp` 印刷）が失敗したら、**プリンターセッション画面に警告として表示**する。
- 直近の警告が分かること（発生時刻・内容）。ユーザーが確認・消去できると望ましい。
- 既存のサーバーログ出力は維持する（消さない）。

### B. 実行時の ON/OFF 切り替え
- プリンターセッション画面に、**自動 PDF / 自動印刷を有効・無効にするトグル**を置く。
- **出力設定が構成されている場合のみ**このトグルを表示する（未設定なら出さない）。
- 切り替えはそのセッションに対して即時に効く（次の受信から反映）。
- 既定は「有効」（従来どおり設定があれば動く）。

## 対象外 / 制約
- プロファイル側の設定内容（autoPdfDir/autoPrint 等）の変更は行わない（別途プロファイル編集で行う）。
  ここでの ON/OFF は**そのセッション実行中の一時的な切り替え**。
- 出力先ディレクトリの自動作成や事前検証は行わない（失敗はエラー表示で気づける形にする）。
- 信頼境界は維持（出力設定の値そのものはブラウザから変更しない。ON/OFF のみ）。

## 完了条件 (受け入れ基準)
- [ ] PDF 保存・自動印刷の失敗がプリンターセッション画面に警告として表示される。
- [ ] 出力設定があるセッションでのみ ON/OFF トグルが表示される（未設定では表示されない）。
- [ ] トグルで無効にすると以後の受信で PDF 保存・自動印刷が行われず、有効に戻すと再開する。
- [ ] 既定は有効で、従来の挙動（設定があれば自動出力）が変わらない。
- [ ] サーバーログの警告出力は従来どおり維持される。

## 未確定事項 / 確認したいこと（spec で解消）
- **Q1（通知の経路）**: 警告を WebSocket の新メッセージで push するか、スプール受信メッセージに載せるか。
  非同期に発生し得るので push が素直（spec で確定）。
- **Q2（ON/OFF の伝達）**: WS のクライアント→サーバーメッセージで切り替えるか、REST を足すか。
  セッション単位の状態なので WS が素直（spec で確定）。
- **Q3（警告の保持）**: セッションに履歴として保持し後から開いた画面にも出すか、都度 push のみか。
