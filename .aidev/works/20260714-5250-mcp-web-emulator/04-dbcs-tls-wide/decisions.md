# 決定記録（04-dbcs-tls-wide）

## D1: DBCS 変換テーブルの複数コードポイント合成エントリはスキップする

- 背景: ibm-1399 の .ucm に `<U00E6><U0300>` のような複数コードポイント合成列 → EBCDIC のマッピングがある。
  単純な cp→bytes の Map では表現できない。
- 決定: gen-tables のパーサはこれらをスキップ（skipped 件数を記録）し、日本語 DBCS の常用文字の
  1:1 マッピングのみをテーブル化する。
- 理由: 合成列は稀で日本語常用文字には影響しない。ibm-1399 で 55 件程度をスキップし 22321 件を採用。
- 影響: なし（常用範囲の DBCS 変換は完全）。

## D2: 実機 PUB400 での DBCS 文字表示検証は限定的（research F3 の制約）

- 背景: PUB400 はプロファイル CCSID の恒久変更不可。日本語入出力は `CHGJOB CCSID(1399)`＋IGCDTA(*YES) の
  自作ソース PF レベルに限られ、日本語 NLV 画面は不可（research F3）。
- 決定: 実機検証は **TLS 接続（992）・DBCS 端末タイプ（IBM-5555-C01）受理・27x132 端末（IBM-3477-FC）受理**の
  確認に留め、**DBCS 文字デコード/描画/桁維持は合成 fixture のリプレイ＋ユニットで検証**する
  （codec 往復・WtdApplier の SO/SI/DBCS セル・format 桁維持・web 2ch・session リプレイ）。
- 理由 / 代替案: 自作ソース PF の作成は本 subtask の範囲外の手作業。合成 trace で全パイプラインを決定的に検証できる。
- 影響: 受け入れ基準「日本語（DBCS）を含む画面が Web・MCP の両方で正しく表示」の**実機での最終確認は
  親の統合 test で、CHGJOB CCSID(1399)＋ソース PF 手順を用意して実施**する（本 subtask は実装と単独検証まで）。

## D4: フィールド入力の忠実化はスコープで分割（04 に core 検証＋SO/SI 表示、web 編集モデルは subtask 05 へ）

- 背景: ユーザーレビューで、フィールド入力の忠実化ギャップが判明（型/コードページ入力制限が未エンフォース、
  web が HTML input 任せで上書きモードでなく PDM/SEU 編集で桁がシフトする、DBCS 入力中の SO/SI 桁未維持、
  SO/SI の {}/表示トグル・キーバインド編集の要望）。
- 決定: **スコープで分割**する。
  - **subtask 04 に追加**（DBCS・core 検証スコープ）: core のフィールド入力内容検証（数値・A/O/J 型・
    コードページ許容文字 → 違反は FIELD_TYPE）と、SO/SI の `{`/`}` 表示トグル。
  - **新規 subtask 05（web フィールド編集・キーボード忠実化）**: web の自前フィールド編集モデル
    （上書き既定・Insert トグル・5250 流バックスペース・DBCS バイト長厳密化・入力中 SO/SI 桁維持）、
    キーバインド編集、カタカナ⇔英小文字の表示トグル。
- 理由: core の入力検証は MCP/Web 両方に効き DBCS フィールド規則の完成でもあるため 04。web の編集モデル
  書き換えはそれ自体が独立した作業単位（HTML input 依存 → 自前編集への転換）のため新 subtask に分離。
- 影響: 親 subtasks に 05 を追加（親 plan を更新）。04 の tasks に T13/T14 を追加。

- 背景: DBCS 端末タイプ（IBM-5555 系）が PUB400 で拒否される懸念（plan リスク）。
- 決定: 実機確認の結果、IBM-5555-C01（DBCS）・IBM-3477-FC（27x132）・IBM-3179-2（既定）いずれも
  PUB400 が受理しメニュー到達。代替経路（3477/3179＋CCSID のみ）は不要だった。
- 影響: なし（当初の端末タイプ表がそのまま有効）。
