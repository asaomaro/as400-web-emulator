# 要件: 認証・per-user 分離と管理者画面

## 背景 / 課題
現状のサーバーは認証を持たず、`prt-1` 等の推測可能な ID ＋無認証エンドポイントにより、共有サーバー運用では
他ユーザーの帳票等を取得できてしまう（前回確認）。商用 web-to-host 相当の**複数ユーザー共有運用**に耐える
認証・per-user 分離が必要。一方で**ローカル単一運用**も残すため、認証は on/off できること。

## 目的 / ゴール
1. **認証（on/off 切替）** と **role（admin/user）** を導入し、ON 時は per-user でセッション・帳票を分離する。
2. **管理者画面**でユーザー管理・エミュレータ/プリンターセッション管理・ログ取得を行えるようにする。
   管理者は top 画面に接続以外のセクションを持ち、クリックで既存同様に**タブ展開**する。

## スコープ（2 段階リリース）
### PR 1 — 認証・分離コア（本作業の第一デリバラブル）
- 認証の on/off（設定）。OFF 時は現状どおり（ローカル運用・無認証）。
- **ローカルユーザーファイル**（JSON、パスワードは Node 標準 scrypt でハッシュ、role=admin/user）。
- ブラウザは**ログイン＋セッション Cookie（httpOnly・署名）**。MCP/自動化は**API トークン**（Authorization: Bearer）。
- **per-user 分離**: セッション（表示/プリンター）に所有者を付与し、WS/HTTP(PDF)/MCP の取得で所有者一致を強制。
  admin は全アクセス可。**プリンターセッション ID を推測不能（randomUUID）に**する。
- 認証 OFF 時は所有者チェックを行わない（挙動不変）。

### PR 2 — 管理者画面（後続作業）
- 管理者 UI（ユーザー CRUD、全セッション一覧/切断、プリンターセッション管理、ログ取得）。
- top 画面に接続以外のセクション（管理者のみ）。クリックで workspace のタブとして展開。

### 対象外
- 外部 IdP/LDAP 連携（将来拡張）。多要素認証。

## 機能要件（PR 1）
- 設定で認証を有効化でき、無効時は現状の無認証動作を保つ。
- ユーザーファイルのユーザーでログインでき、セッション Cookie が発行される。
- 各ユーザーに API トークンを持たせ、MCP/HTTP を Bearer で認証できる。
- 認証有効時、あるユーザーは自分が開いたセッション/帳票のみ取得でき、他ユーザーのものは拒否される（admin 除く）。
- プリンターセッション ID が推測不能。

## 非機能要件 / 制約
- **セキュリティ**: パスワードは平文保存しない（scrypt ハッシュ＋salt）。Cookie は httpOnly＋署名。
  トークンは十分な乱数長。タイミング安全な比較。ユーザーファイル・秘密はコミットしない（`.gitignore`）。
- 既存の表示/プリンター/PDF 経路を壊さない（認証 OFF で完全後方互換）。
- 依存追加は最小（可能なら Node 標準 crypto で完結）。
